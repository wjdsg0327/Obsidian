# C++：Windows 内存管理专题

Windows 内存 API 主要分为：虚拟内存、堆、内存映射文件、共享内存、进程内存读写、内存保护、系统内存状态、AWE/大页/NUMA 等。

> 官方入口：
> - Memory Management Reference：https://learn.microsoft.com/windows/win32/memory/memory-management-reference
> - Memory Management Functions：https://learn.microsoft.com/windows/win32/memory/memory-management-functions
> - Virtual Memory Functions：https://learn.microsoft.com/windows/win32/memory/virtual-memory-functions
> - Heap Functions：https://learn.microsoft.com/windows/win32/memory/heap-functions

## 1. 内存 API 分类

| 分类 | 典型 API | 用途 | 常见场景 |
|---|---|---|---|
| 虚拟内存 | `VirtualAlloc` / `VirtualFree` | 保留、提交、释放进程虚拟地址空间 | 大块内存、JIT、手动页管理 |
| 页面保护 | `VirtualProtect` / `VirtualQuery` | 改内存页权限、查询页面状态 | DEP、JIT、调试、内存扫描 |
| 跨进程内存 | `VirtualAllocEx` / `ReadProcessMemory` / `WriteProcessMemory` | 操作另一个进程的地址空间 | 调试器、注入器、自动化工具 |
| 堆 | `GetProcessHeap` / `HeapAlloc` / `HeapFree` | 从进程堆或私有堆分配小块内存 | 高频小对象、插件隔离分配 |
| 低碎片堆 | `HeapSetInformation` | 启用 Low Fragmentation Heap | 大量不同大小小块分配 |
| 内存映射文件 | `CreateFileMapping` / `MapViewOfFile` / `UnmapViewOfFile` | 把文件或页文件映射进地址空间 | 大文件读写、共享内存 |
| 共享内存 | `CreateFileMapping(INVALID_HANDLE_VALUE, ...)` | 基于页文件的命名共享内存 | 进程间通信 IPC |
| 系统内存状态 | `GlobalMemoryStatusEx` / `GetPhysicallyInstalledSystemMemory` | 查询物理内存、虚拟内存、页面文件 | 诊断、监控、资源限制 |
| 工作集 | `GetProcessMemoryInfo` / `SetProcessWorkingSetSize` | 查询/调整进程工作集 | 性能诊断，不建议滥调 |
| 安全清零 | `SecureZeroMemory` | 防止编译器优化掉敏感数据清零 | 密码、密钥、Token |
| 大页 | `GetLargePageMinimum` + `VirtualAlloc(MEM_LARGE_PAGES)` | 使用 large page | 数据库、低延迟系统，需要权限 |
| AWE | `AllocateUserPhysicalPages` / `MapUserPhysicalPages` | 地址窗口扩展 | 老式大内存场景，现在较少用 |
| NUMA | `VirtualAllocExNuma` | 指定 NUMA 节点 | 多路服务器性能优化 |

## 2. 核心概念

### Reserve 与 Commit

`VirtualAlloc` 可以分两步：

- `MEM_RESERVE`：保留一段虚拟地址范围，不消耗实际物理存储。
- `MEM_COMMIT`：提交页面，让这段地址真正可访问，背后可由 RAM 或页面文件支持。

常见组合：

```cpp
void* p = VirtualAlloc(nullptr, size, MEM_RESERVE | MEM_COMMIT, PAGE_READWRITE);
```

释放时：

```cpp
VirtualFree(p, 0, MEM_RELEASE);
```

如果只是释放物理提交但保留地址范围：

```cpp
VirtualFree(p, size, MEM_DECOMMIT);
```

### 页面权限

常见保护标志：

| 标志 | 含义 |
|---|---|
| `PAGE_NOACCESS` | 不可访问 |
| `PAGE_READONLY` | 只读 |
| `PAGE_READWRITE` | 读写 |
| `PAGE_EXECUTE` | 只执行 |
| `PAGE_EXECUTE_READ` | 执行 + 读 |
| `PAGE_EXECUTE_READWRITE` | 执行 + 读写，不推荐长期使用 |
| `PAGE_GUARD` | 保护页，首次访问触发异常 |

安全建议：如果要 JIT 或动态代码，常见流程是：

1. `VirtualAlloc(..., PAGE_READWRITE)` 写入机器码。
2. `VirtualProtect(..., PAGE_EXECUTE_READ)` 改成可执行不可写。
3. 不要长期保持 `PAGE_EXECUTE_READWRITE`。

## 3. 常用 API 详解

### `VirtualAlloc`

```cpp
LPVOID VirtualAlloc(
  LPVOID lpAddress,
  SIZE_T dwSize,
  DWORD  flAllocationType,
  DWORD  flProtect
);
```

用途：在当前进程地址空间保留/提交页面。

常用参数：

- `lpAddress = nullptr`：让系统选择地址。
- `flAllocationType = MEM_RESERVE | MEM_COMMIT`。
- `flProtect = PAGE_READWRITE`。

### `VirtualFree`

```cpp
BOOL VirtualFree(LPVOID lpAddress, SIZE_T dwSize, DWORD dwFreeType);
```

- `MEM_RELEASE`：`dwSize` 必须是 0。
- `MEM_DECOMMIT`：释放提交，地址仍保留。

### `VirtualProtect`

改已提交页面的访问权限。

```cpp
DWORD oldProtect = 0;
VirtualProtect(p, size, PAGE_READONLY, &oldProtect);
```

### `VirtualQuery`

查询某地址所在内存区域的状态。

```cpp
MEMORY_BASIC_INFORMATION mbi{};
VirtualQuery(p, &mbi, sizeof(mbi));
```

`MEMORY_BASIC_INFORMATION` 关键字段：

| 字段 | 含义 |
|---|---|
| `BaseAddress` | 区域基址 |
| `AllocationBase` | 初始分配基址 |
| `AllocationProtect` | 初始保护 |
| `RegionSize` | 区域大小 |
| `State` | `MEM_COMMIT` / `MEM_RESERVE` / `MEM_FREE` |
| `Protect` | 当前页保护 |
| `Type` | `MEM_PRIVATE` / `MEM_MAPPED` / `MEM_IMAGE` |

### `HeapAlloc` / `HeapFree`

用于堆内存。默认堆：

```cpp
HANDLE heap = GetProcessHeap();
void* p = HeapAlloc(heap, HEAP_ZERO_MEMORY, 1024);
HeapFree(heap, 0, p);
```

私有堆：

```cpp
HANDLE heap = HeapCreate(0, 1024 * 1024, 0);
void* p = HeapAlloc(heap, 0, 4096);
HeapFree(heap, 0, p);
HeapDestroy(heap);
```

适合大量小块、生命周期相近、想集中释放或隔离碎片的场景。

### `CreateFileMapping` / `MapViewOfFile`

用于文件映射或共享内存。

文件映射：

```cpp
HANDLE file = CreateFileW(L"data.bin", GENERIC_READ | GENERIC_WRITE, 0, nullptr, OPEN_EXISTING, 0, nullptr);
HANDLE mapping = CreateFileMappingW(file, nullptr, PAGE_READWRITE, 0, 0, nullptr);
void* view = MapViewOfFile(mapping, FILE_MAP_ALL_ACCESS, 0, 0, 0);

// 使用 view 访问文件内容

UnmapViewOfFile(view);
CloseHandle(mapping);
CloseHandle(file);
```

共享内存：

```cpp
HANDLE mapping = CreateFileMappingW(
    INVALID_HANDLE_VALUE,
    nullptr,
    PAGE_READWRITE,
    0,
    4096,
    L"Local\\MySharedMemory");
void* view = MapViewOfFile(mapping, FILE_MAP_ALL_ACCESS, 0, 0, 4096);
```

另一个进程用 `OpenFileMappingW` 打开同名对象。

### `ReadProcessMemory` / `WriteProcessMemory`

用于调试器、内存查看器、自动化工具等。需要目标进程句柄有权限。

```cpp
HANDLE h = OpenProcess(PROCESS_VM_READ | PROCESS_QUERY_INFORMATION, FALSE, pid);
char buf[128]{};
SIZE_T read = 0;
ReadProcessMemory(h, address, buf, sizeof(buf), &read);
CloseHandle(h);
```

写入远程进程通常需要：

- `PROCESS_VM_OPERATION`
- `PROCESS_VM_WRITE`
- `PROCESS_CREATE_THREAD`（如果还要远程线程）

注意：这类能力容易触发安全软件，且涉及权限边界。

### `GlobalMemoryStatusEx`

查询系统内存状态：

```cpp
MEMORYSTATUSEX s{};
s.dwLength = sizeof(s);
if (GlobalMemoryStatusEx(&s)) {
    wprintf(L"Memory load: %u%%\n", s.dwMemoryLoad);
    wprintf(L"Total phys: %llu MB\n", s.ullTotalPhys / 1024 / 1024);
    wprintf(L"Avail phys: %llu MB\n", s.ullAvailPhys / 1024 / 1024);
}
```

## 4. 案例：VirtualAlloc 分配并保护内存

```cpp
#include <windows.h>
#include <iostream>

int wmain() {
    const SIZE_T size = 4096;

    void* p = VirtualAlloc(nullptr, size, MEM_RESERVE | MEM_COMMIT, PAGE_READWRITE);
    if (!p) {
        std::wcerr << L"VirtualAlloc failed: " << GetLastError() << L"\n";
        return 1;
    }

    auto bytes = static_cast<unsigned char*>(p);
    bytes[0] = 0x41;
    bytes[1] = 0x42;

    DWORD oldProtect = 0;
    if (!VirtualProtect(p, size, PAGE_READONLY, &oldProtect)) {
        std::wcerr << L"VirtualProtect failed: " << GetLastError() << L"\n";
        VirtualFree(p, 0, MEM_RELEASE);
        return 1;
    }

    MEMORY_BASIC_INFORMATION mbi{};
    if (VirtualQuery(p, &mbi, sizeof(mbi))) {
        std::wcout << L"Region size: " << mbi.RegionSize << L"\n";
        std::wcout << L"Protect: " << mbi.Protect << L"\n";
    }

    VirtualFree(p, 0, MEM_RELEASE);
    return 0;
}
```

## 5. 案例：文件映射读取大文件

```cpp
#include <windows.h>
#include <iostream>

int wmain() {
    HANDLE file = CreateFileW(L"data.bin", GENERIC_READ, FILE_SHARE_READ, nullptr,
                              OPEN_EXISTING, FILE_ATTRIBUTE_NORMAL, nullptr);
    if (file == INVALID_HANDLE_VALUE) return 1;

    HANDLE mapping = CreateFileMappingW(file, nullptr, PAGE_READONLY, 0, 0, nullptr);
    if (!mapping) { CloseHandle(file); return 1; }

    void* view = MapViewOfFile(mapping, FILE_MAP_READ, 0, 0, 0);
    if (!view) {
        CloseHandle(mapping);
        CloseHandle(file);
        return 1;
    }

    // view 指向文件内容，适合随机访问大文件。

    UnmapViewOfFile(view);
    CloseHandle(mapping);
    CloseHandle(file);
    return 0;
}
```

## 6. 选择建议

| 需求 | 推荐方式 |
|---|---|
| 普通 C++ 对象 | `new/delete`、STL 容器、智能指针 |
| 大块页级内存 | `VirtualAlloc` |
| 频繁小块分配 | 默认堆/私有堆，或 C++ allocator |
| 共享内存 IPC | `CreateFileMapping` + `MapViewOfFile` |
| 大文件随机读取 | 文件映射 |
| 敏感数据清零 | `SecureZeroMemory` |
| 查询系统内存 | `GlobalMemoryStatusEx` |
| 调试/读其他进程 | `ReadProcessMemory` |

## 7. 常见坑

- `VirtualFree(..., MEM_RELEASE)` 时 size 必须是 `0`。
- `VirtualAlloc` 返回的大小按页面粒度，地址对齐也按系统粒度。
- `PAGE_EXECUTE_READWRITE` 不要长期使用，容易违反 W^X 原则。
- `HeapAlloc` 得来的内存不能用 `free/delete` 释放。
- `new/delete`、`malloc/free`、`HeapAlloc/HeapFree`、`LocalAlloc/LocalFree` 不能混用。
- 映射视图用 `UnmapViewOfFile`，映射对象句柄用 `CloseHandle`。
- `ReadProcessMemory` 不是万能的，目标地址可能不可读，跨位数/权限/UAC 都会影响。
- 敏感数据用 `SecureZeroMemory`，普通 `memset` 可能被优化掉。
