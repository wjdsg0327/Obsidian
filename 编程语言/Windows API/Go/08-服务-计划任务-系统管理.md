# Go：服务、计划任务、系统管理

Windows Service、SCM、事件日志、WMI、组策略、性能计数器。

## Go 调用方式

| API | 用途 | 库/模块 |
|---|---|---|
| `OpenSCManager` | 打开服务控制管理器 | golang.org/x/sys/windows 或 syscall/DLL |
| `CreateService/OpenService` | 创建/打开服务 | golang.org/x/sys/windows 或 syscall/DLL |
| `StartService/ControlService` | 启动/控制服务 | go-ole 或 syscall + unsafe |
| `QueryServiceStatusEx` | 查询服务状态 | golang.org/x/sys/windows 或 syscall/DLL |
| `RegisterServiceCtrlHandlerEx` | 服务控制回调 | golang.org/x/sys/windows 或 syscall/DLL |
| `StartServiceCtrlDispatcher` | 服务入口调度 | golang.org/x/sys/windows 或 syscall/DLL |
| `OpenEventLog/ReportEvent` | 旧事件日志 | golang.org/x/sys/windows 或 syscall/DLL |
| `EvtQuery/EvtNext` | Windows Event Log | golang.org/x/sys/windows 或 syscall/DLL |
| `PdhOpenQuery/PdhAddCounter` | 性能计数器 | go-ole 或 syscall + unsafe |
| `CoCreateInstance(IWbemLocator)` | WMI COM 入口 | go-ole 或 syscall + unsafe |

## 使用要点

- 优先使用 Go 标准库；标准库不够时再用 `golang.org/x/sys/windows`。
- Windows 字符串通常是 UTF-16：`windows.UTF16PtrFromString`、`windows.UTF16ToString`。
- 结构体传参要注意字段类型、对齐、`unsafe.Sizeof`。
- `HANDLE`、证书上下文、COM 内存、注册表 Key 等都要使用对应释放函数。
- `syscall.NewLazyDLL` 适合补充未封装 API，但要按官方签名核对参数宽度。

## 案例

```go
// Go 调 Windows API 通用骨架：
// 1. 优先查 golang.org/x/sys/windows 是否已经封装
// 2. 字符串转 UTF-16：windows.UTF16PtrFromString
// 3. 调用 API，检查 err / 返回值
// 4. defer windows.CloseHandle 或调用对应释放函数
```
