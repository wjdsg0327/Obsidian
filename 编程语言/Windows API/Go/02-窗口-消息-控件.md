# Go：窗口、消息、控件

窗口类、消息循环、窗口过程、菜单、资源、通用控件、DWM、高 DPI。

## Go 调用方式

| API | 用途 | 库/模块 |
|---|---|---|
| `RegisterClassEx` | 注册窗口类 | golang.org/x/sys/windows 或 syscall/DLL |
| `CreateWindowEx` | 创建窗口 | golang.org/x/sys/windows 或 syscall/DLL |
| `ShowWindow` | 显示窗口 | golang.org/x/sys/windows 或 syscall/DLL |
| `UpdateWindow` | 刷新窗口 | golang.org/x/sys/windows 或 syscall/DLL |
| `GetMessage/TranslateMessage/DispatchMessage` | 消息循环 | golang.org/x/sys/windows 或 syscall/DLL |
| `DefWindowProc` | 默认窗口过程 | golang.org/x/sys/windows 或 syscall/DLL |
| `SendMessage/PostMessage` | 同步/异步发送消息 | golang.org/x/sys/windows 或 syscall/DLL |
| `SetWindowLongPtr/GetWindowLongPtr` | 保存窗口关联数据 | golang.org/x/sys/windows 或 syscall/DLL |
| `CreateDialog/DialogBox` | 对话框 | golang.org/x/sys/windows 或 syscall/DLL |
| `InitCommonControlsEx` | 初始化通用控件 | go-ole 或 syscall + unsafe |
| `DwmSetWindowAttribute` | DWM 属性 | golang.org/x/sys/windows 或 syscall/DLL |
| `SetProcessDpiAwarenessContext` | DPI 感知 | go-ole 或 syscall + unsafe |

## 使用要点

- 优先使用 Go 标准库；标准库不够时再用 `golang.org/x/sys/windows`。
- Windows 字符串通常是 UTF-16：`windows.UTF16PtrFromString`、`windows.UTF16ToString`。
- 结构体传参要注意字段类型、对齐、`unsafe.Sizeof`。
- `HANDLE`、证书上下文、COM 内存、注册表 Key 等都要使用对应释放函数。
- `syscall.NewLazyDLL` 适合补充未封装 API，但要按官方签名核对参数宽度。

## 案例

```go
// Go 调 Windows API 通用骨架：
// 1. 优先查 golang.org/x/sys/windows 是否已经封装
// 2. 字符串转 UTF-16：windows.UTF16PtrFromString
// 3. 调用 API，检查 err / 返回值
// 4. defer windows.CloseHandle 或调用对应释放函数
```
