# Go：文件、目录、磁盘、I/O

文件句柄、同步/异步 I/O、目录、卷、路径、内存映射文件。

## Go 调用方式

| API | 用途 | 库/模块 |
|---|---|---|
| `CreateFile` | 打开/创建文件、设备、管道 | golang.org/x/sys/windows 或 syscall/DLL |
| `ReadFile/WriteFile` | 读写文件或设备 | golang.org/x/sys/windows 或 syscall/DLL |
| `CloseHandle` | 关闭句柄 | golang.org/x/sys/windows 或 syscall/DLL |
| `GetFileAttributesEx` | 读取文件属性 | golang.org/x/sys/windows 或 syscall/DLL |
| `FindFirstFile/FindNextFile` | 遍历目录 | golang.org/x/sys/windows 或 syscall/DLL |
| `CreateDirectory/RemoveDirectory` | 创建/删除目录 | golang.org/x/sys/windows 或 syscall/DLL |
| `DeleteFile/MoveFileEx/CopyFile` | 文件操作 | go-ole 或 syscall + unsafe |
| `GetFileSizeEx/GetFileTime` | 大小和时间 | golang.org/x/sys/windows 或 syscall/DLL |
| `CreateFileMapping/MapViewOfFile` | 内存映射文件 | golang.org/x/sys/windows 或 syscall/DLL |
| `DeviceIoControl` | 设备控制与磁盘查询 | go-ole 或 syscall + unsafe |
| `GetDiskFreeSpaceEx` | 磁盘空间 | golang.org/x/sys/windows 或 syscall/DLL |
| `GetVolumeInformation` | 卷信息 | golang.org/x/sys/windows 或 syscall/DLL |

## 使用要点

- 优先使用 Go 标准库；标准库不够时再用 `golang.org/x/sys/windows`。
- Windows 字符串通常是 UTF-16：`windows.UTF16PtrFromString`、`windows.UTF16ToString`。
- 结构体传参要注意字段类型、对齐、`unsafe.Sizeof`。
- `HANDLE`、证书上下文、COM 内存、注册表 Key 等都要使用对应释放函数。
- `syscall.NewLazyDLL` 适合补充未封装 API，但要按官方签名核对参数宽度。

## 案例

```go
//go:build windows
package main

import (
    "fmt"
    "golang.org/x/sys/windows"
)

func main() {
    p, _ := windows.UTF16PtrFromString("demo.txt")
    h, err := windows.CreateFile(p, windows.GENERIC_WRITE, 0, nil,
        windows.CREATE_ALWAYS, windows.FILE_ATTRIBUTE_NORMAL, 0)
    if err != nil { panic(err) }
    defer windows.CloseHandle(h)
    var written uint32
    data := []byte("hello windows api from go
")
    err = windows.WriteFile(h, data, &written, nil)
    fmt.Println(written, err)
}
```
