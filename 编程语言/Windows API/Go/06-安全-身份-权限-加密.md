# Go：安全、身份、权限、加密

Token、SID、ACL、权限提升、认证授权、CryptoAPI/CNG、证书、DPAPI。

## Go 调用方式

| API | 用途 | 库/模块 |
|---|---|---|
| `OpenProcessToken` | 打开进程令牌 | golang.org/x/sys/windows 或 syscall/DLL |
| `LookupPrivilegeValue` | 查找权限 LUID | golang.org/x/sys/windows 或 syscall/DLL |
| `AdjustTokenPrivileges` | 启用/禁用权限 | golang.org/x/sys/windows 或 syscall/DLL |
| `GetTokenInformation` | 查询 Token | golang.org/x/sys/windows 或 syscall/DLL |
| `ConvertSidToStringSid` | SID 转字符串 | go-ole 或 syscall + unsafe |
| `InitializeSecurityDescriptor` | 安全描述符 | golang.org/x/sys/windows 或 syscall/DLL |
| `AccessCheck` | 访问检查 | golang.org/x/sys/windows 或 syscall/DLL |
| `CryptProtectData/CryptUnprotectData` | DPAPI 加解密 | golang.org/x/sys/windows 或 syscall/DLL |
| `BCryptOpenAlgorithmProvider` | CNG 算法 | golang.org/x/sys/windows 或 syscall/DLL |
| `CertOpenSystemStore/CertEnumCertificatesInStore` | 证书存储 | golang.org/x/sys/windows 或 syscall/DLL |

## 使用要点

- 优先使用 Go 标准库；标准库不够时再用 `golang.org/x/sys/windows`。
- Windows 字符串通常是 UTF-16：`windows.UTF16PtrFromString`、`windows.UTF16ToString`。
- 结构体传参要注意字段类型、对齐、`unsafe.Sizeof`。
- `HANDLE`、证书上下文、COM 内存、注册表 Key 等都要使用对应释放函数。
- `syscall.NewLazyDLL` 适合补充未封装 API，但要按官方签名核对参数宽度。

## 案例

```go
// Go 调 Windows API 通用骨架：
// 1. 优先查 golang.org/x/sys/windows 是否已经封装
// 2. 字符串转 UTF-16：windows.UTF16PtrFromString
// 3. 调用 API，检查 err / 返回值
// 4. defer windows.CloseHandle 或调用对应释放函数
```
