# Go：图形、音视频、多媒体

GDI/GDI+、Direct2D/DirectWrite、Direct3D、WIC、Media Foundation、Core Audio。

## Go 调用方式

| API | 用途 | 库/模块 |
|---|---|---|
| `BeginPaint/EndPaint` | 窗口绘制 | golang.org/x/sys/windows 或 syscall/DLL |
| `GetDC/ReleaseDC` | 设备上下文 | golang.org/x/sys/windows 或 syscall/DLL |
| `CreatePen/CreateBrush/DeleteObject` | GDI 对象 | golang.org/x/sys/windows 或 syscall/DLL |
| `BitBlt/StretchBlt` | 位图拷贝 | golang.org/x/sys/windows 或 syscall/DLL |
| `GdiplusStartup` | GDI+ 初始化 | golang.org/x/sys/windows 或 syscall/DLL |
| `D2D1CreateFactory` | Direct2D 工厂 | golang.org/x/sys/windows 或 syscall/DLL |
| `DWriteCreateFactory` | DirectWrite | golang.org/x/sys/windows 或 syscall/DLL |
| `CoCreateInstance(IMF*)` | Media Foundation | go-ole 或 syscall + unsafe |
| `CoCreateInstance(MMDeviceEnumerator)` | Core Audio | go-ole 或 syscall + unsafe |
| `WICCreateImagingFactory` | WIC 图像编解码 | golang.org/x/sys/windows 或 syscall/DLL |

## 使用要点

- 优先使用 Go 标准库；标准库不够时再用 `golang.org/x/sys/windows`。
- Windows 字符串通常是 UTF-16：`windows.UTF16PtrFromString`、`windows.UTF16ToString`。
- 结构体传参要注意字段类型、对齐、`unsafe.Sizeof`。
- `HANDLE`、证书上下文、COM 内存、注册表 Key 等都要使用对应释放函数。
- `syscall.NewLazyDLL` 适合补充未封装 API，但要按官方签名核对参数宽度。

## 案例

```go
// Go 调 Windows API 通用骨架：
// 1. 优先查 golang.org/x/sys/windows 是否已经封装
// 2. 字符串转 UTF-16：windows.UTF16PtrFromString
// 3. 调用 API，检查 err / 返回值
// 4. defer windows.CloseHandle 或调用对应释放函数
```
