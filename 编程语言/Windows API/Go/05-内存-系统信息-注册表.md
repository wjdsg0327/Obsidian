# Go：内存、系统信息、注册表

虚拟内存、堆、系统版本、计算机名、时间、注册表、对象和句柄。

## Go 调用方式

| API | 用途 | 库/模块 |
|---|---|---|
| `VirtualAlloc/VirtualFree` | 虚拟内存 | golang.org/x/sys/windows 或 syscall/DLL |
| `HeapCreate/HeapAlloc/HeapFree` | 堆内存 | golang.org/x/sys/windows 或 syscall/DLL |
| `GlobalMemoryStatusEx` | 内存状态 | golang.org/x/sys/windows 或 syscall/DLL |
| `GetSystemInfo/GetNativeSystemInfo` | CPU/页大小 | golang.org/x/sys/windows 或 syscall/DLL |
| `GetComputerNameEx` | 计算机名 | go-ole 或 syscall + unsafe |
| `GetVersionEx/RtlGetVersion` | 系统版本 | golang.org/x/sys/windows 或 syscall/DLL |
| `GetTickCount64` | 开机毫秒数 | go-ole 或 syscall + unsafe |
| `QueryPerformanceCounter` | 高精度计时 | go-ole 或 syscall + unsafe |
| `RegOpenKeyEx/RegQueryValueEx` | 读取注册表 | golang.org/x/sys/windows 或 syscall/DLL |
| `RegSetValueEx/RegCloseKey` | 写入/关闭注册表 | golang.org/x/sys/windows 或 syscall/DLL |

## 使用要点

- 优先使用 Go 标准库；标准库不够时再用 `golang.org/x/sys/windows`。
- Windows 字符串通常是 UTF-16：`windows.UTF16PtrFromString`、`windows.UTF16ToString`。
- 结构体传参要注意字段类型、对齐、`unsafe.Sizeof`。
- `HANDLE`、证书上下文、COM 内存、注册表 Key 等都要使用对应释放函数。
- `syscall.NewLazyDLL` 适合补充未封装 API，但要按官方签名核对参数宽度。

## 案例

```go
//go:build windows
package main

import (
    "fmt"
    "golang.org/x/sys/windows"
)

func main() {
    var info windows.SystemInfo
    windows.GetNativeSystemInfo(&info)
    fmt.Println("page size:", info.PageSize)
    fmt.Println("processors:", info.NumberOfProcessors)
}
```
