# 配置与工程化

## tsconfig.json 基础

### 初始化配置

```bash
# 生成 tsconfig.json
tsc --init
```

### 基础配置

```json
{
  "compilerOptions": {
    // 编译目标
    "target": "ES2020",
    "module": "ESNext",
    "lib": ["ES2020", "DOM", "DOM.Iterable"],
    
    // 输出
    "outDir": "./dist",
    "rootDir": "./src",
    
    // 严格模式
    "strict": true,
    
    // 模块解析
    "moduleResolution": "node",
    "esModuleInterop": true,
    "allowSyntheticDefaultImports": true,
    
    // 路径
    "baseUrl": ".",
    "paths": {
      "@/*": ["src/*"]
    },
    
    // 其他
    "skipLibCheck": true,
    "forceConsistentCasingInFileNames": true,
    "resolveJsonModule": true,
    "declaration": true,
    "declarationMap": true,
    "sourceMap": true
  },
  "include": ["src/**/*"],
  "exclude": ["node_modules", "dist"]
}
```

## 编译选项详解

### target

```json
{
  "compilerOptions": {
    // ES5 - 最大兼容性
    "target": "ES5",
    
    // ES2015/ES6 - 支持 Promise, Map, Set
    "target": "ES2015",
    
    // ES2020 - 支持 Optional Chaining, Nullish Coalescing
    "target": "ES2020",
    
    // ESNext - 最新特性
    "target": "ESNext"
  }
}
```

### module

```json
{
  "compilerOptions": {
    // CommonJS - Node.js 默认
    "module": "CommonJS",
    
    // ESNext - 现代打包工具
    "module": "ESNext",
    
    // ES2015 - import/export
    "module": "ES2015"
  }
}
```

### strict 模式

```json
{
  "compilerOptions": {
    // 开启所有严格检查
    "strict": true,
    
    // 等价于单独开启：
    "strictNullChecks": true,
    "strictFunctionTypes": true,
    "strictBindCallApply": true,
    "strictPropertyInitialization": true,
    "noImplicitAny": true,
    "noImplicitThis": true,
    "alwaysStrict": true
  }
}
```

### 路径配置

```json
{
  "compilerOptions": {
    "baseUrl": ".",
    "paths": {
      "@/*": ["src/*"],
      "@components/*": ["src/components/*"],
      "@utils/*": ["src/utils/*"]
    }
  }
}
```

## 项目结构

### 基础结构

```
project/
├── src/
│   ├── index.ts
│   ├── models/
│   │   └── user.ts
│   ├── services/
│   │   └── api.ts
│   └── utils/
│       └── format.ts
├── dist/              # 编译输出
├── tsconfig.json
├── package.json
└── node_modules/
```

### 分离配置

```
project/
├── src/
├── tsconfig.json          # 基础配置
├── tsconfig.build.json    # 构建配置
├── tsconfig.test.json     # 测试配置
└── package.json
```

tsconfig.build.json：
```json
{
  "extends": "./tsconfig.json",
  "compilerOptions": {
    "outDir": "./dist",
    "declaration": true,
    "sourceMap": false
  },
  "exclude": ["**/*.test.ts", "**/*.spec.ts"]
}
```

tsconfig.test.json：
```json
{
  "extends": "./tsconfig.json",
  "compilerOptions": {
    "sourceMap": true
  },
  "include": ["**/*.test.ts", "**/*.spec.ts"]
}
```

## 构建工具集成

### esbuild

```bash
npm install -D esbuild
```

package.json：
```json
{
  "scripts": {
    "build": "esbuild src/index.ts --bundle --outdir=dist --format=esm",
    "build:watch": "esbuild src/index.ts --bundle --outdir=dist --format=esm --watch"
  }
}
```

### Vite

```bash
npm create vite@latest my-app -- --template vanilla-ts
```

vite.config.ts：
```typescript
import { defineConfig } from "vite"

export default defineConfig({
  build: {
    target: "es2020",
    outDir: "dist"
  }
})
```

### Rollup

```bash
npm install -D rollup @rollup/plugin-typescript
```

rollup.config.ts：
```typescript
import typescript from "@rollup/plugin-typescript"

export default {
  input: "src/index.ts",
  output: {
    dir: "dist",
    format: "esm"
  },
  plugins: [typescript()]
}
```

### Webpack

```bash
npm install -D webpack webpack-cli ts-loader
```

webpack.config.js：
```javascript
module.exports = {
  entry: "./src/index.ts",
  module: {
    rules: [
      {
        test: /\.tsx?$/,
        use: "ts-loader",
        exclude: /node_modules/
      }
    ]
  },
  resolve: {
    extensions: [".tsx", ".ts", ".js"]
  },
  output: {
    filename: "bundle.js",
    path: __dirname + "/dist"
  }
}
```

## 调试配置

### VS Code 调试

.vscode/launch.json：
```json
{
  "version": "0.2.0",
  "configurations": [
    {
      "type": "node",
      "request": "launch",
      "name": "Debug TypeScript",
      "program": "${workspaceFolder}/src/index.ts",
      "preLaunchTask": "tsc: build - tsconfig.json",
      "outFiles": ["${workspaceFolder}/dist/**/*.js"]
    }
  ]
}
```

### ts-node 直接运行

```bash
# 安装
npm install -D ts-node

# 运行
npx ts-node src/index.ts

# 使用 tsconfig-paths 支持路径别名
npm install -D tsconfig-paths
npx ts-node -r tsconfig-paths/register src/index.ts
```

### tsx（更快的替代品）

```bash
# 安装
npm install -D tsx

# 运行
npx tsx src/index.ts

# 监听模式
npx tsx watch src/index.ts
```

## 常见问题

### 路径别名不生效

```bash
# 安装 tsconfig-paths
npm install -D tsconfig-paths

# 运行时
npx ts-node -r tsconfig-paths/register src/index.ts

# 或者使用 tsx
npx tsx src/index.ts  # 自动支持路径别名
```

### 循环依赖

```typescript
// 使用 import type 避免
import type { User } from "./user"
```

### 第三方库缺少类型

```bash
# 安装类型
npm install -D @types/lodash

# 或者声明
declare module "some-lib" {
  export function doSomething(): void
}
```
