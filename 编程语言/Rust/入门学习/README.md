## 学习目标

学完以后你应该能：

- 看懂 Rust 项目的基本结构：`Cargo.toml`、`src/main.rs`、`src/lib.rs`
- 会用 `cargo new/run/build/test` 创建、运行、测试项目
- 理解 Rust 最核心的所有权、借用、可变引用
- 会用 `String`、`Vec`、`HashMap` 等常用类型
- 会写 `struct`、`enum`、`match`
- 会处理 `Option` 和 `Result`
- 会查找并使用第三方库，例如 `serde`、`tokio`、`clap`
- 能独立写一个简单 CLI 小项目

## 建议节奏

- 第 1 天：01-03，能跑起来 Rust 程序
- 第 2 天：04，重点啃所有权，不急着快
- 第 3 天：05-07，开始写实际代码
- 第 4 天：08-09，理解项目组织和 Rust 风格
- 第 5 天：10，做一个完整小项目

## Rust 学习时的心态

Rust 一开始难，不是因为语法多，而是因为它把“内存安全”和“并发安全”这些以前运行时才暴露的问题，提前放到了编译期。

所以新手最常见的感觉是：

> 我明明知道我要干什么，但编译器不让我写。

这很正常。把 Rust 编译器当成一个严格的代码老师，而不是敌人。
