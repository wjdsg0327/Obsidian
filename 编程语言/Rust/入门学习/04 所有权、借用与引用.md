# 04 所有权、借用与引用

这是 Rust 最核心、也是最容易卡住的一章。

不要急着背生命周期，先把所有权和借用理解清楚。

## 为什么 Rust 需要所有权

很多语言管理内存的方式：

- C/C++：程序员手动申请和释放，容易忘记释放、重复释放、野指针
- Java/C#/Go：垃圾回收 GC 自动释放，简单但有运行时成本
- Rust：编译期检查所有权，到点自动释放，不需要 GC

Rust 的目标：

> 既要 C/C++ 的性能，又要内存安全。

## 所有权三条规则

Rust 所有权规则：

1. Rust 中每个值都有一个所有者 owner
2. 同一时间只能有一个所有者
3. 当所有者离开作用域，值会被释放

例子：

```rust
fn main() {
    let s = String::from("hello");
    println!("{}", s);
} // s 离开作用域，内存自动释放
```

## move：所有权转移

```rust
fn main() {
    let s1 = String::from("hello");
    let s2 = s1;

    // println!("{}", s1); // 报错
    println!("{}", s2);
}
```

`s1` 的所有权移动给了 `s2`，之后 `s1` 不能再用。

为什么？

`String` 的内容在堆上。如果简单复制指针，两个变量都以为自己拥有这块内存，最后可能释放两次。Rust 直接禁止这种情况。

## clone：真正复制

如果确实想复制一份：

```rust
fn main() {
    let s1 = String::from("hello");
    let s2 = s1.clone();

    println!("s1 = {}, s2 = {}", s1, s2);
}
```

`clone` 会复制堆上的数据，有成本。

## Copy 类型

简单类型通常会自动复制：

```rust
fn main() {
    let x = 5;
    let y = x;

    println!("x = {}, y = {}", x, y);
}
```

因为 `i32` 这种类型存在栈上，复制成本很低。

常见 `Copy` 类型：

- 整数
- 浮点数
- bool
- char
- 只包含 Copy 类型的 tuple

## 函数也会拿走所有权

```rust
fn take(s: String) {
    println!("{}", s);
}

fn main() {
    let s = String::from("hello");
    take(s);
    // println!("{}", s); // 报错，s 已经被移动
}
```

如果不想函数拿走所有权，就用引用。

## 引用：借用值但不拿走

```rust
fn print_len(s: &String) {
    println!("长度: {}", s.len());
}

fn main() {
    let s = String::from("hello");
    print_len(&s);
    println!("还能用: {}", s);
}
```

`&s` 表示借用 `s`。

函数参数 `&String` 表示接收一个引用。

## 更推荐用 &str

对于只读字符串，通常写：

```rust
fn print_len(s: &str) {
    println!("长度: {}", s.len());
}

fn main() {
    let s = String::from("hello");
    print_len(&s);
    print_len("world");
}
```

`&str` 更灵活，既能接收字符串字面量，也能接收 `String` 的引用。

## 可变引用

如果要通过引用修改值：

```rust
fn add_world(s: &mut String) {
    s.push_str(" world");
}

fn main() {
    let mut s = String::from("hello");
    add_world(&mut s);
    println!("{}", s);
}
```

注意两边都要写 `mut`：

- 变量本身 `let mut s`
- 传参时 `&mut s`
- 函数参数 `&mut String`

## 借用规则

Rust 最重要的借用规则：

1. 同一时间，可以有多个不可变引用
2. 同一时间，只能有一个可变引用
3. 不可变引用和可变引用不能同时存在

可以：

```rust
let s = String::from("hello");
let r1 = &s;
let r2 = &s;
println!("{}, {}", r1, r2);
```

不可以：

```rust
let mut s = String::from("hello");
let r1 = &s;
let r2 = &mut s; // 报错
println!("{}", r1);
```

原因：防止一边读一边改，造成不安全。

## 引用的作用域

现在 Rust 编译器比较聪明。引用最后一次使用后，作用域就可以结束：

```rust
let mut s = String::from("hello");
let r1 = &s;
println!("{}", r1);

let r2 = &mut s;
r2.push_str(" world");
```

因为 `r1` 在 `println!` 后不再使用，所以后面可以创建可变引用。

## 悬垂引用会被禁止

下面代码不允许：

```rust
fn dangle() -> &String {
    let s = String::from("hello");
    &s
}
```

因为 `s` 离开函数后会被释放，返回它的引用就变成野指针。

正确做法是返回所有权：

```rust
fn no_dangle() -> String {
    let s = String::from("hello");
    s
}
```

## 新手判断口诀

遇到所有权问题时，先问：

1. 这个值谁拥有？
2. 我是要拿走它，还是只看一眼？
3. 如果只看，用 `&T`
4. 如果要改，用 `&mut T`
5. 如果要复制，用 `clone()`，但别滥用

## 本节练习

修复下面代码：

```rust
fn print_name(name: String) {
    println!("{}", name);
}

fn main() {
    let name = String::from("老王");
    print_name(name);
    println!("再次使用: {}", name);
}
```

参考答案：

```rust
fn print_name(name: &str) {
    println!("{}", name);
}

fn main() {
    let name = String::from("老王");
    print_name(&name);
    println!("再次使用: {}", name);
}
```

## 本节小结

所有权是 Rust 的核心。新手阶段先掌握：

- `String` 赋值通常会 move
- `clone()` 才是真复制
- 函数参数会影响所有权
- 只读传 `&T` 或 `&str`
- 修改传 `&mut T`
