# C++ 快速学习笔记总目录

> 目标：用最短路径从 C++ 基础语法学到能做项目、能接入常见框架。

## 学习说明

C++ 不建议只背语法。最快的路线是：

```text
基础语法 -> 指针/引用/内存 -> 面向对象 -> STL -> 现代 C++ -> CMake 工程化 -> 框架/项目实战
```

如果你已经有 Python / Go / Java / C# 基础，可以把重点放在 **内存模型、RAII、STL、现代 C++、构建系统** 这几块。

## 最快上手路线

如果你想尽快“能写东西”，按这个顺序：

1. 第 1 天：编译运行、变量、输入输出、流程控制
2. 第 2 天：函数、数组、字符串、`vector`
3. 第 3 天：指针、引用、栈/堆、对象生命周期
4. 第 4 天：类、构造/析构、封装、继承、多态
5. 第 5 天：STL 容器：`vector`、`map`、`unordered_map`、`set`
6. 第 6 天：STL 算法、Lambda、范围 `for`
7. 第 7 天：智能指针、RAII、异常处理、文件读写
8. 第 8~10 天：CMake、项目结构、第三方库管理
9. 第 11~14 天：选一个框架做小项目

## 框架方向怎么选

| 目标 | 推荐框架/库 | 适合项目 |
|---|---|---|
| 图像处理 / 视觉 AI | OpenCV | 人脸检测、车道线检测、OCR 前处理 |
| 桌面软件 | Qt | 工具软件、管理后台、跨平台 GUI |
| 调试工具 / 内部工具 UI | Dear ImGui | 游戏工具、实时参数面板 |
| 小游戏 / 图形入门 | raylib / SFML | 2D 游戏、可视化 Demo |
| Web 后端 API | Crow / Drogon | REST API、小型服务 |
| 网络 / 异步 IO | Boost.Asio | TCP/UDP 服务、网络工具 |

建议第一条实战路线选 **OpenCV**：你的知识库里已经有完整的 [OpenCV项目实战指南](OpenCV项目实战指南.md)，能从语言学习自然过渡到项目。

## 一句话建议

学 C++ 的关键不是把所有语法一次性背完，而是尽快建立这几个意识：**对象生命周期、资源所有权、标准库优先、少手写裸指针、用 CMake 管项目**。
