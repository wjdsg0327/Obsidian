TypeScript 入门资料：语法、函数与热门库

面向有一点 JavaScript 基础的学习路线：先理解“类型系统是在开发阶段帮你抓 bug”，再逐步进入函数、接口、泛型、工程化和常用库。

为什么学 TypeScript

TypeScript 可以理解为“带类型的 JavaScript”。浏览器和 Node.js 最终运行的仍然是 JavaScript，TypeScript 主要在开发阶段通过编译器和编辑器提示，提前发现变量写错、函数参数不对、对象字段缺失等问题。

核心心法：TypeScript 不是新运行时，而是 JavaScript 的静态类型层。写 TS 时，始终要想“这个值可能是什么形状”。

环境与项目结构

最常见的学习环境是 Node.js + npm/pnpm + TypeScript 编译器。前端项目通常用 Vite，后端项目可以用 tsx 或 ts-node 开发。

npm init -y
npm install -D typescript tsx @types/node
npx tsc --init
# 运行单个 ts 文件
npx tsx src/index.ts

常见目录：

project/
  src/
    index.ts
  package.json
  tsconfig.json
常见坑：没有安装 @types/node 时，Node 的 process、Buffer 等类型可能报错。

基础类型与类型推断

基础类型包括 string、number、boolean、null、undefined、数组、元组和字面量类型。很多时候 TS 会自动推断，不必每行都写类型。

let name = 'Lihua';      // 推断为 string
let age: number = 18;
let tags: string[] = ['前端', '工具'];
let point: [number, number] = [120, 30];
let status: 'draft' | 'published' = 'draft';
建议开启 strict: true。一开始报错会多，但能逼你养成正确的类型习惯。

函数、参数与返回值

函数是 TypeScript 最常用的类型入口。要关注三件事：参数类型、返回值类型、可选/默认参数。

function add(a: number, b: number): number {
  return a + b;
}

function greet(name: string, title = '同学'): string {
  return `${title}，${name}`;
}

function log(message: string): void {
  console.log(message);
}

函数也可以作为值传递：

type Mapper = (value: number) => string;

const toPrice: Mapper = (value) => `¥${value.toFixed(2)}`;
常见坑：不要滥用 any。如果暂时不知道类型，优先用 unknown，再通过判断收窄。

对象、接口与类型别名

对象类型描述“这个对象有哪些字段、字段分别是什么类型”。接口 interface 和类型别名 type 都能做这件事。

interface User {
  id: number;
  name: string;
  email?: string; // 可选字段
}

type Article = {
  title: string;
  author: User;
  tags: string[];
};

简单建议：描述对象结构优先用 interface；联合类型、工具组合、函数类型常用 type。

联合类型、收窄与枚举

联合类型表示“可能是 A，也可能是 B”。用 if、typeof、in、判别字段等方式，可以把宽类型收窄成具体类型。

type Result =
  | { ok: true; data: string }
  | { ok: false; error: string };

function handle(result: Result) {
  if (result.ok) {
    console.log(result.data);
  } else {
    console.error(result.error);
  }
}
实际项目里，判别联合类型非常适合描述接口返回、任务状态、表单状态。

泛型与常用工具类型

泛型可以把类型变成参数，让函数或数据结构保持灵活，同时不丢失类型信息。

function first<T>(items: T[]): T | undefined {
  return items[0];
}

const n = first([1, 2, 3]);      // number | undefined
const s = first(['a', 'b']);     // string | undefined

常用工具类型：

Partial<T>
把字段都变成可选
Required<T>
把字段都变成必填
Pick<T, K>
挑选部分字段
Omit<T, K>
排除部分字段
Record<K, V>
构造键值映射
ReturnType<F>
获取函数返回值类型

模块、配置与工程实践

现代 TypeScript 使用 ES Module：export 导出，import 导入。工程上重点看 tsconfig.json。

// math.ts
export function add(a: number, b: number) {
  return a + b;
}

// index.ts
import { add } from './math';

推荐配置方向：

{
  "compilerOptions": {
    "target": "ES2022",
    "module": "ESNext",
    "moduleResolution": "Bundler",
    "strict": true,
    "skipLibCheck": true
  }
}
常见坑：前端、Node、打包工具的模块配置不同。遇到 import/export 报错时，先检查 module、moduleResolution、type: module。

练习路线与资料

  1. 第 1 阶段：把 JS 基础变量、数组、对象、函数全部改写成 TS。
  2. 第 2 阶段：写一个命令行 Todo：新增、完成、删除、保存 JSON。
  3. 第 3 阶段：写一个前端页面：表单 + 列表 + 类型化 API 数据。
  4. 第 4 阶段：用 Zod 校验接口返回，再把类型传给业务代码。
  5. 第 5 阶段：做一个小后端：Hono/NestJS + Prisma/Drizzle + Vitest。

推荐资料

  • TypeScript 官方文档：https://www.typescriptlang.org/docs/
  • TypeScript Handbook：官方手册，最系统。
  • TypeScript Playground：在线试类型推断和编译结果。
  • Total TypeScript：进阶练习很好。
  • Type Challenges：刷类型体操，适合进阶，不建议一开始就刷。
学习顺序建议：先会写业务类型，再学泛型；先会看报错，再追求高级类型。TS 最实用的价值是让项目更稳，不是炫类型体操。
