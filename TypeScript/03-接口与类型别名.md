# 接口与类型别名

## 接口（Interface）

### 基础接口

```typescript
interface User {
  name: string
  age: number
  email: string
}

const user: User = {
  name: "张三",
  age: 25,
  email: "zhangsan@example.com"
}
```

### 可选属性

```typescript
interface User {
  name: string
  age: number
  email?: string  // 可选
}

const user1: User = { name: "张三", age: 25 }
const user2: User = { name: "李四", age: 30, email: "lisi@example.com" }
```

### 只读属性

```typescript
interface User {
  readonly id: number
  name: string
}

const user: User = { id: 1, name: "张三" }
// user.id = 2  // 错误！只读属性不能修改
```

### 函数类型

```typescript
interface SearchFunc {
  (source: string, subString: string): boolean
}

const mySearch: SearchFunc = (source, subString) => {
  return source.includes(subString)
}
```

### 索引签名

```typescript
interface StringArray {
  [index: number]: string
}

const arr: StringArray = {
  0: "第一个",
  1: "第二个"
}

interface Dictionary {
  [key: string]: string
}

const dict: Dictionary = {
  name: "张三",
  city: "北京"
}
```

## 接口继承

### 单继承

```typescript
interface Animal {
  name: string
  age: number
}

interface Dog extends Animal {
  breed: string
}

const dog: Dog = {
  name: "旺财",
  age: 3,
  breed: "金毛"
}
```

### 多继承

```typescript
interface CanRun {
  run(): void
}

interface CanSwim {
  swim(): void
}

interface SuperAnimal extends CanRun, CanSwim {
  name: string
}

const duck: SuperAnimal = {
  name: "唐老鸭",
  run() { console.log("跑步") },
  swim() { console.log("游泳") }
}
```

## 类型别名（Type Alias）

### 基础用法

```typescript
// 简单类型别名
type ID = string | number
type Name = string

// 对象类型别名
type User = {
  id: ID
  name: Name
  email: string
}

// 函数类型别名
type Callback = (data: string) => void
type AsyncCallback = (data: string) => Promise<void>
```

### 联合类型

```typescript
type Status = "active" | "inactive" | "pending"
type Result = Success | Error
type ID = string | number
```

### 交叉类型

```typescript
type Person = {
  name: string
  age: number
}

type Worker = {
  company: string
  position: string
}

type Employee = Person & Worker

const emp: Employee = {
  name: "张三",
  age: 25,
  company: "ABC",
  position: "工程师"
}
```

## Interface vs Type

### 相同点

```typescript
// 都可以描述对象
interface IUser {
  name: string
}

type TUser = {
  name: string
}

// 都可以继承
interface IChild extends IUser {
  age: number
}

type TChild = TUser & {
  age: number
}
```

### 不同点

| 特性 | Interface | Type |
|------|-----------|------|
| 声明合并 | ✅ 支持 | ❌ 不支持 |
| 联合类型 | ❌ 不支持 | ✅ 支持 |
| 原始类型别名 | ❌ 不支持 | ✅ 支持 |
| 元组 | ❌ 不支持 | ✅ 支持 |
| 继承方式 | extends | & |

### 声明合并（Interface 独有）

```typescript
interface User {
  name: string
}

interface User {
  age: number
}

// 合并后：{ name: string; age: number }
const user: User = {
  name: "张三",
  age: 25
}
```

### Type 独有能力

```typescript
// 联合类型
type Status = "active" | "inactive"

// 原始类型别名
type ID = string | number

// 元组
type Pair = [string, number]

// 条件类型
type IsString<T> = T extends string ? true : false
```

---

## 使用建议

| 场景 | 推荐 |
|------|------|
| 定义对象结构 | Interface |
| 需要声明合并 | Interface |
| 联合/交叉类型 | Type |
| 原始类型别名 | Type |
| 函数类型 | Type（更简洁） |
| 工具类型 | Type |

## 扩展接口与类型

### 扩展接口

```typescript
interface BaseConfig {
  host: string
  port: number
}

interface AppConfig extends BaseConfig {
  debug: boolean
  logLevel: string
}

// 也可以用交叉类型
type AppConfig2 = BaseConfig & {
  debug: boolean
  logLevel: string
}
```

### 泛型接口

```typescript
interface ApiResponse<T> {
  code: number
  message: string
  data: T
}

interface User {
  id: number
  name: string
}

const response: ApiResponse<User> = {
  code: 200,
  message: "success",
  data: { id: 1, name: "张三" }
}
```

## 实战示例

### API 接口定义

```typescript
// 请求参数
interface GetUserParams {
  id: number
}

// 响应数据
interface UserInfo {
  id: number
  name: string
  email: string
  avatar?: string
}

// API 响应
interface ApiResponse<T> {
  code: number
  message: string
  data: T
}

// 函数类型
type GetUserApi = (params: GetUserParams) => Promise<ApiResponse<UserInfo>>

// 实现
const getUser: GetUserApi = async (params) => {
  const response = await fetch(`/api/users/${params.id}`)
  return response.json()
}
```

### 表单配置

```typescript
interface FormField {
  name: string
  label: string
  type: "input" | "select" | "textarea"
  required?: boolean
  placeholder?: string
  options?: Array<{ label: string; value: string | number }>
}

interface FormConfig {
  fields: FormField[]
  submitText?: string
  cancelText?: string
}

const userForm: FormConfig = {
  fields: [
    { name: "name", label: "姓名", type: "input", required: true },
    { name: "role", label: "角色", type: "select", options: [
      { label: "管理员", value: "admin" },
      { label: "用户", value: "user" }
    ]}
  ],
  submitText: "提交"
}
```
