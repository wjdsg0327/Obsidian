Godot 图文入门教学

这份教程适合刚接触 Godot 的学习者。目标不是把所有 API 背下来，而是先建立正确的游戏开发思维：游戏由场景组成，场景由节点组成，节点用脚本和信号协作。

01 Godot 是什么

Godot 是一个免费、开源、跨平台的游戏引擎，可以做 2D、3D、XR 项目。它的特色是轻量、编辑器启动快、2D 工作流舒服，并且围绕“节点 + 场景”的组合方式组织游戏。

Godot 在游戏开发中的位置

适合做什么

  • 2D 游戏：平台跳跃、弹幕、RPG、塔防、解谜、独立小游戏。
  • 3D 游戏：小中型 3D、原型验证、低多边形风格项目。
  • 工具与交互应用：可视化工具、教学互动、简单编辑器。
常见误区

不要一开始就纠结“Godot、Unity、Unreal 哪个最强”。入门阶段更重要的是学会游戏对象如何组织、逻辑如何更新、事件如何流动。Godot 的结构清晰，特别适合先建立这些基本功。

02 安装与项目结构

Godot 通常不需要复杂安装：下载编辑器，解压或直接运行即可。建议从 Godot 4.x 开始学习。新建项目时，每个项目就是一个独立文件夹，里面保存场景、脚本、素材和项目配置。

推荐项目目录结构

新建项目建议

  1. 项目路径不要放在太深、太乱、含奇怪符号的目录里。
  2. 中文路径一般可用，但跨平台协作时英文目录更稳。
  3. 刚开始就分好 scenes、scripts、assets，以后项目不容易乱。
小建议

学习 Godot 时先做 2D。2D 项目能更快理解节点、脚本、信号、碰撞、UI，之后再转 3D 会顺很多。

03 编辑器界面

Godot 编辑器可以理解为一个“场景搭建器”。左边通常是场景树和文件系统，中间是 2D/3D 视图，右边是属性检查器，底部是输出、调试、动画等面板。

Godot 编辑器区域示意

最常用的四个地方

Scene 场景树

查看当前场景由哪些节点组成，节点的父子关系也在这里调整。

Inspector 检查器

修改节点属性，例如位置、颜色、碰撞形状、脚本变量。

FileSystem 文件系统

管理素材、场景、脚本。拖拽资源到视口或节点上很常用。

Viewport 视口

直接摆放角色、按钮、碰撞区域、摄像机等对象。

常见误区

不要把 Godot 当成“只写代码的框架”。它是可视化编辑器 + 脚本系统。很多事情应该在编辑器中搭结构、配属性，再用脚本写行为。

04 节点与场景：Godot 的核心

Godot 官方文档把核心概念概括为：游戏是节点树，节点组合成场景，场景再组合成完整游戏；节点之间可以用信号通信。理解这句话，Godot 就入门一半了。

节点组成场景，场景组成游戏

节点 Node

节点是最小功能单元。比如 Sprite2D 负责显示图片，CollisionShape2D 负责碰撞形状，Camera2D 负责镜头。

场景 Scene

场景是一棵保存下来的节点树。角色可以是一个场景，敌人可以是一个场景，UI 面板也可以是一个场景。场景可以被重复实例化，例如生成 20 个敌人。

练习

新建一个 Player.tscn：根节点用 CharacterBody2D，下面加 Sprite2D 和 CollisionShape2D。这就是最常见的 2D 玩家结构。

05 GDScript 基础

GDScript 是 Godot 内置脚本语言，语法接近 Python，但它不是 Python。它专门为 Godot 设计，和节点、场景、编辑器集成很紧。

脚本挂在节点上，控制节点行为

一个基础移动脚本大概长这样：

extends CharacterBody2D

@export var speed: float = 300.0

func _physics_process(delta):
    var direction := Vector2.ZERO

    if Input.is_action_pressed("move_right"):
        direction.x += 1
    if Input.is_action_pressed("move_left"):
        direction.x -= 1
    if Input.is_action_pressed("move_down"):
        direction.y += 1
    if Input.is_action_pressed("move_up"):
        direction.y -= 1

    velocity = direction.normalized() * speed
    move_and_slide()

几个必须认识的点

  • extends CharacterBody2D：说明这个脚本是给 CharacterBody2D 节点用的。
  • @export：把变量暴露到检查器里，可以不改代码就调速度。
  • _physics_process(delta)：固定物理帧调用，适合移动和碰撞。
  • move_and_slide()：Godot 提供的角色移动方法，会处理滑动碰撞。
常见误区

初学者容易把所有代码都塞进一个脚本。更好的方式是：Player 管玩家，Enemy 管敌人，HUD 管界面，Main 负责把它们组织起来。

06 输入、移动与碰撞

玩家移动通常涉及三件事：读取输入、计算速度、交给物理系统移动。碰撞则需要合适的物理节点和碰撞形状。

从按键到角色移动的流程

InputMap 的好处

不要在代码里直接写“按 W 键”。更推荐在 Project Settings → Input Map 里创建 move_up、move_down 这些动作，再把按键绑定到动作上。这样以后改键位不需要改代码。

碰撞节点选择

  • CharacterBody2D：玩家、敌人这类主动移动角色。
  • Area2D：检测进入、拾取、伤害区域、触发器。
  • StaticBody2D：墙、地面、不可移动障碍物。
  • RigidBody2D：受物理影响的箱子、球、掉落物。
记忆方法

玩家会自己走，用 CharacterBody2D；只是检测有没有进入，用 Area2D；固定不动挡路，用 StaticBody2D。

07 信号机制：让对象低耦合通信

信号是 Godot 的事件系统。按钮被按下、计时器超时、玩家进入区域、敌人死亡，都可以发出信号。其他节点连接这个信号后，就能在事件发生时执行函数。

信号：事件从一个节点传到另一个节点

示例：玩家死亡时发出信号，让主场景处理游戏结束。

# Player.gd
signal died

func take_damage():
    died.emit()

# Main.gd
func _ready():
    $Player.died.connect(_on_player_died)

func _on_player_died():
    print("游戏结束")

为什么用信号

如果 Player 直接去控制 HUD、Main、Enemy，代码会越写越乱。用信号后，Player 只负责宣布“我死了”，谁关心这个事件，谁自己连接。

常见误区

信号不是为了炫技。简单父子节点直接调用方法也可以；当两个节点不适合互相知道对方存在时，再用信号会更清爽。

08 做一个 2D 小游戏：躲避敌人

官方入门项目叫 “Dodge the Creeps!”：玩家移动躲避不断出现的敌人，存活越久分数越高。这是很好的第一个完整项目，因为它覆盖了 2D 游戏的主要基础。

小游戏结构：Main 组织所有子系统

制作步骤

  1. 创建 Player 场景：根节点 CharacterBody2D，添加 Sprite2D 和 CollisionShape2D。
  2. 写玩家移动：用 InputMap 读取方向，设置 velocity，调用 move_and_slide。
  3. 创建 Enemy 场景：可以用 Area2D 或 CharacterBody2D，根据需求移动。
  4. 创建 Main 场景：把 Player、HUD、Timer 放进去。
  5. 随机刷敌：Timer timeout 后实例化 Enemy，并设置随机位置和方向。
  6. 碰撞失败：玩家碰到敌人后发出 died 信号，Main 停止游戏。
  7. 计分与重开：HUD 显示分数，按钮控制开始新游戏。
最小可玩目标

不要一开始追求美术和特效。第一个版本只要做到：玩家能移动、敌人能出现、碰到敌人会结束、分数会增加，就已经是完整游戏循环。

09 资源、导出与学习路线

Godot 学习最有效的路线是：先完成小项目，再补知识点。只看文档容易记不住，只做项目又容易卡住；两者结合最快。

推荐学习路线

30 天学习安排

  • 第 1 周：熟悉编辑器、节点、场景、GDScript 基础。
  • 第 2 周：做完一个 2D 躲避小游戏，理解输入、碰撞、Timer、信号。
  • 第 3 周：学习 UI、动画、音效、资源导入和简单状态机。
  • 第 4 周：做一个自己的小项目：例如打砖块、塔防原型、平台跳跃或背包系统。

导出发布

Godot 可以导出到 Windows、Linux、macOS、Web、Android 等平台。第一次导出时需要安装导出模板，然后在 Project → Export 中配置平台。

常见误区

不要等“学完 Godot”才开始做游戏。游戏引擎永远学不完。正确方式是先做一个小得不能再小的完整游戏，再逐步加功能。
