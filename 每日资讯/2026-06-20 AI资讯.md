## Codex 更新

### 官方文档确认的能力面

OpenAI 帮助中心文章《Using Codex with your ChatGPT plan》显示：

- Codex 是帮助写、审、发布代码的 AI agent；
- 可通过以下入口使用：
  - Codex app
  - Codex CLI
  - Codex IDE extension
  - Codex web
- Codex 覆盖 Free、Go、Plus、Pro、Business、Edu、Enterprise 等计划，使用限制与 credit 选项按计划不同；
- 企业设置包括：
  - Plugins：Business / Enterprise / Edu workspace 可由管理员控制插件；
  - Sites in Codex：Business / Enterprise workspace 预览，用于创建并部署 workspace 内部 web apps；
  - Browser use / in-app browser Developer mode：可启用 CDP 访问做更深的浏览器调试，但涉及敏感能力，需要显式批准与管理员策略；
  - RBAC：按角色授予 Codex 访问；
  - Compliance API：覆盖 Codex 使用日志，包括 CLI、IDE extension、Web / Cloud delegated usage。

### 第三方/媒体提到的近期 Codex 方向

VentureBeat 与 Totalum 等第三方报道提到 OpenAI 在 6 月初围绕 Codex 推出：

- Sites：面向企业的内部网页/工作区部署；
- Annotations：局部上下文/局部编辑能力，减少整文档重写风险；
- 角色插件：面向销售、产品设计、分析、创意、财务等业务流程；
- Codex Computer Use / Windows 桌面控制相关能力。

## CLIProxyAPI 更新

固定仓库：`router-for-me/CLIProxyAPI`

### 最新版本

- **最新 Release / Tag：v7.2.21**
- 发布时间：GitHub Releases 显示为 **2026-06-19 16:59**
- 对比：`v7.2.20...v7.2.21`

### v7.2.21 变更要点

- `feat(translator)`：在 OpenAI response conversion 中把 `reasoning_content` 附加到 assistant 和 tool messages；
- `chore(deps)`：升级 `github.com/jackc/pgx/v5` 从 v5.7.6 到 v5.9.2；
- `feat(translator)`：在 OpenAI request conversions 中保留 `input_audio` fields；
- `feat(translator)`：Codex function call starts 延迟到 function name 可用；
- `feat(schema)`：JSON schema 处理中移除 `$comment` 和 `enumDescriptions`；
- `feat(auth)`：统一 provider key handling，支持 OpenAI compatibility provider key 处理。

### 近两版值得注意

- **v7.2.20（2026-06-19 06:44）**：移除 Gemini CLI provider 及相关 translator packages / logic；保留 structured `tool_choice`、`input_image` details；
- **v7.2.19（2026-06-18 21:35）**：改进 config reload，引入 async management save hook；OpenAI responses 中 drop `apply_patch` custom tool；Codex requests 中 normalize `service_tier`；支持 namespace tools；新增 transient error cooldown 与 file-backed persistent cooldown state。

### 判断

CLIProxyAPI 最近更新非常密集，重点集中在：

- Codex / OpenAI Responses 兼容性；
- multimodal fields（audio/image）保留；
- auth / provider key / cooldown 稳定性；
- schema 与 tool conversion 的细节兼容。

如果本地使用 CLIProxyAPI 做 Codex / OpenAI-compatible gateway，建议重点关注 v7.2.19 → v7.2.21 的兼容性变化，尤其是 Gemini CLI provider 移除与 Codex function call 处理。

## OpenClaw 更新

### 官方文档概况

OpenClaw 官方文档首页介绍：OpenClaw 是自托管 multi-channel gateway，可连接 Discord、Google Chat、iMessage、Matrix、Microsoft Teams、Signal、Slack、Telegram、WhatsApp、Zalo 等渠道到 AI agent；推荐 Node 24，Node 22 LTS 22.19+ 可兼容。

### 最新 Release

GitHub Releases 显示：

- **2026.6.9**
- 发布时间：**2026-06-19 05:52**
- 状态：**Pre-release**

### 2026.6.9 Highlights

- **Telegram delivery 增强**：Telegram 支持 rich HTML，保留 rich markdown / sticker paths，更好渲染 progress drafts 与 command output，mentions 与 spooled handlers 走正确交付路径；
- **Agent recovery 更可靠**：重试、终端结果、compaction 后 usage、session history repair、reply reconciliation 等让中断/部分完成的 turn 更容易走向可见最终结果；
- **Codex integration 增强**：Codex 自动 plugin approvals、GPT-5.3 Spark OAuth routing、remote-node exec 作为 dynamic tool、更可靠的 app-server teardown 与 terminal outcomes；
- **官方 provider plugins 独立化**：external provider packages 成为 first-class npm releases，外部 channel plugins 在 Gateway 启动时加载；
- **客户端增强**：Control UI 增加 session workspace rail 与 extension health；iOS 增加 Watch controls；Android 显示 chat context；
- **搜索与技能**：Codex Hosted Search 可用；key-free search providers 保持 opt-in；ClawHub skill installs 保留 verified source provenance。

### 判断

这版更像“可靠性 + 交付体验 + Codex/插件生态”的综合增强。由于是 Pre-release，生产环境升级前建议等 stable 或先在测试环境验证 QQ/Telegram/Codex/插件加载链路。

## Hermes Agent 更新

固定仓库：`NousResearch/hermes-agent`

### 最新版本

- **Hermes Agent v0.17.0 / v2026.6.19**
- Release Date：**2026-06-19**
- GitHub Release 时间：**2026-06-19 19:39**
- 代号：**The Reach Release**

Release notes 称：自 v0.16.0 以来约 1,475 commits、约 800 merged PRs、1,693 files changed、300+ issues closed、245 community contributors。

### v0.17.0 重点

- **iMessage via Photon Spectrum**：Hermes 增加 iMessage platform plugin，使用 Photon managed line pool，无需 Mac relay / BlueBubbles 自托管桥；
- **Raft agent network**：新增 bundled Raft platform adapter，可作为外部 agent 通过 wake-channel bridge 接入 Raft，release notes 强调 wake payload 只携带 metadata；
- **桌面端大幅增强**：可重绑快捷键、原生 OS notifications、subagent watch-windows、composer model selector、RTL/bidi 自动方向、VS Code-themed terminal pane、per-thread composer drafts、安装 VS Code Marketplace theme；
- **后台/异步 subagents**：`delegate_task(background=true)` 可立即返回 handle，subagent 后台运行，完成后结果回到对话；
- **图片编辑**：`image_generate` 支持 image-to-image 编辑/转换；
- **Automation Blueprints**：不用手写 cron，通过命名自动化与表单/对话收集参数创建计划任务；
- **Cursor Composer via xAI Grok**：`grok-composer-2.5-fast` 出现在 xAI OAuth model picker，context window 调整为 200k；
- **Dashboard profile builder**：浏览器里创建完整 Hermes profile，选择 model、skills、MCP servers；
- **Skills Hub 重做**：connected hubs、Featured、安装前预览、安全扫描；
- **Memory tool 原子批量操作**：通过 operations array 做 add/replace/remove 批量变更，减少多轮编辑脆弱性。

### Tags / 历史版本

- `v2026.6.19`：Hermes Agent v0.17.0，2026-06-19；
- `v2026.6.5`：Hermes Agent v0.16.0，The Surface Release，2026-06-06，重点为 native desktop app、browser admin panel、remote-gateway connect、简中桌面 UI、leaner default skills、NVIDIA/skills trusted tap、fuzzy model picker、`/undo`。

### 判断

Hermes Agent 这次更新跨度大，重点从“桌面可用”扩展到“多渠道触达 + 团队/浏览器管理 + 后台代理 + 媒体编辑”。如果老王后续对比 OpenClaw / Hermes，可重点看：Hermes 的学习闭环、后台 subagent 与 Dashboard profile builder；OpenClaw 则继续强在多渠道 gateway 与 QQ/Telegram/节点生态。
