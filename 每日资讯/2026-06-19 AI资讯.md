## Codex 更新

### 官方/较可信更新

#### 1. OpenAI：用 Codex 构建“自改进税务 agent”闭环（2026-05-27）

OpenAI 官方工程文章介绍了 Thrive Holdings 与 OpenAI 如何把真实业务反馈转成 Codex 可处理的工程任务：

1. 从税务从业者的人工修正中捕捉可行动问题；
2. 用 product traces 将修正转成 eval；
3. 将 eval 和 repo、只读生产证据、任务环境一起交给 Codex，生成候选修复并通过回归验证。

文章强调：Codex 不是直接“自动上线改动”，而是在有边界的 writable worktree、明确 eval、只读生产上下文和人工 review 的环境里工作。

- 重点意义：这是 OpenAI 把 Codex 从“写代码工具”推进到“生产系统持续改进组件”的典型案例。
- 来源：OpenAI — Building self-improving tax agents with Codex

#### 2. OpenAI 推出面向白领工作的 Codex 新工具（媒体报道，2026-06-02）

TechCrunch 报道称，OpenAI 为 Codex 推出新的企业/白领工作流相关能力：

- **Codex Sites**：让 Codex 输出托管的交互式网站，而不只是本地文件；合作生态提到 Wix、Base44、Replit、Lovable、Figma、Emergent 等。
- **Annotations**：允许用户指定文档/文件中的具体部分，提供更精准的上下文和命令范围。
- **插件支持/企业集成**：报道称 OpenAI 在 3 月引入 Codex 插件支持，并持续扩展企业应用。

- 备注：该项来自媒体报道，建议以后补查 OpenAI 官方 Codex release notes 或产品页面确认细节。
- 来源：TechCrunch — OpenAI launches new Codex tools for white-collar work

### 社区反馈/风险观察

OpenAI Developer Community 近期出现多条 Codex 相关反馈，主要包括：

- GPT-5.5 xhigh / Codex CLI 出现 “Selected model is at capacity” 容量提示；
- 部分用户反馈 6 月中旬后 Codex 限额消耗变快；
- 桌面 app 与移动端连接、Windows + WSL 性能、模型下线/可用性等问题。

这些是用户反馈，不代表官方结论。对老王的实际使用建议：

- 长任务优先保留本地 Git checkpoint；
- 遇到 capacity 时先降 effort（xhigh → high/medium）或换模型；
- Windows + WSL 场景可优先尝试直接在 WSL 内跑 CLI，减少桌面 app 跨 `/mnt/c` 扫描的影响。

## Claude 更新

### 1. Anthropic 发布 Claude Opus 4.8（官方）

Anthropic 官方公告发布 **Claude Opus 4.8**，价格与 Opus 4.7 常规用法保持一致：

- 常规：$5 / 百万 input tokens，$25 / 百万 output tokens；
- fast mode：$10 / 百万 input tokens，$50 / 百万 output tokens；
- API 模型名：`claude-opus-4-8`。

官方定位为“相对 Opus 4.7 modest but tangible improvement”，重点是更好的协作、编码任务表现和更谨慎/诚实的错误暴露。

- 来源：Anthropic — Introducing Claude Opus 4.8

### 2. Claude Code：Dynamic workflows 研究预览

随 Opus 4.8 一起发布的重点能力：**dynamic workflows**（研究预览）。Anthropic 表示 Claude Code 可以：

- 规划大型任务；
- 在单个 session 中运行数百个并行 subagents；
- 在回报用户前验证输出；
- 用于大规模代码迁移等“代码库级”任务。

可用范围：Claude Code 的 Enterprise、Team、Max plans。

### 3. Effort control 与 Claude Code `xhigh`

Anthropic 表示 Opus 4.8 默认 high effort；用户可选择：

- `extra`（Claude Code 中对应 `xhigh`）；
- `max`；
- 更低 effort 以换取更快响应和更慢消耗限额。

Claude Code rate limits 已为更高 effort 的 token 用量做过上调。

### 4. Messages API 支持 messages 数组中的 system entries

Anthropic 官方说明，Messages API 现在接受 messages 数组里的 system entries。用途包括：

- agent 运行中更新权限；
- 更新 token budget；
- 更新环境上下文；
- 不破坏 prompt cache，也不必通过 user turn 注入。

这对长期 agent harness 和动态权限管理比较重要。

### 5. Claude Design 与 Claude Code 联动（媒体报道）

CNET 报道，Claude Design 更新后可与 Claude Code 双向联动：

- 管理员可设置设计系统/品牌 kit；
- Claude 可检查品牌合规；
- 可导入 GitHub repo、设计文件和上传素材；
- Claude Code 可通过 `/design` 命令拉取设计内容，设计与代码两侧保持同步。

- 备注：此项来自媒体报道，建议后续补查 Anthropic 官方产品页或 release notes。

### v2026.6.8 重点

- **UI / mobile session 更稳**：workspace files 默认折叠、WebChat backscroll 保留、iOS stale foreground Gateway 重连；
- **memory / state 更韧性**：OpenAI embedding batch 超限前拆分、QMD search transient mode 可用、SQLite 避免 NFS WAL 等。

### v0.16.0 / The Surface Release 重点

根据 GitHub release：

- **Hermes Desktop 原生桌面 app**：Electron 应用，支持 macOS/Linux/Windows，一键安装、应用内自更新、拖拽文件、剪贴板图片、Cmd+K 命令面板、状态栏模型选择等；
- **远程 Hermes gateway**：桌面 app 可连接远程 gateway，支持 OAuth 或用户名/密码；
- **Web dashboard 升级为完整 admin panel**：可在浏览器中管理 MCP catalog、消息 channel、credentials、webhooks、memory、gateway controls、Debug Share 等；
- **简体中文 UI**：桌面 app 完整简体中文界面；
- **默认 skills 瘦身**：移除冗余/废弃技能，重型或小众技能改为可选安装，并引入 environment relevance gate；
- **模型选择更好用**：desktop/web/TUI/CLI 的 model picker 支持 fuzzy search；
- **`/undo`**：支持撤回最近 N 轮；
- **安全修复**：Starlette CVE-2026-48710 pin、SSRF off-loop hardening、subprocess credential stripping 等。
