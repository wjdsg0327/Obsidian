## 开发生态

### [Anthropic上调Claude API速率限制并简化层级](https://x.com/ClaudeDevs/status/2072818299361263778) `#1`

> Anthropic 近日宣布为所有用户提高 Claude Platform 上 API 的调用频率上限，并简化了层级结构，不再基于 API 使用量分级。

Anthropic 官方表示，Claude 平台 API 的速率限制已整体提升，并且层级设计不再依赖 API 使用量来划分。对开发者来说，这意味着在 Opus、Sonnet、Haiku 之间切换时，速率限制差异减少，模型选型可以更偏向任务需求，而不是被吞吐额度卡住。官方还提到，最新 Sonnet 和 Haiku 在最高层级可获得 5 倍更高的速率限制。

**影响判断：** 这是典型的“开发者体验优化”更新，对做高并发调用、Agent 编排和企业接入的人更友好。

### [社区报告 Claude Code 疑似跨会话缓存泄漏，Agent输出无关内容](https://github.com/anthropics/claude-code/issues/74066) `#2`

> 据用户报告，Claude Code 出现疑似跨会话缓存泄漏，Agent 突然谈论 Minecraft 等无关内容。社区讨论认为可能是“缓存隔离缺陷”或模型幻觉，Anthropic 尚未公开回应。

GitHub Issue #74066 中，用户反馈 Claude Code 在长时间响应后突然输出与当前任务无关的内容，甚至疑似混入别的会话信息。讨论中存在两种主要解释：一种认为这是缓存隔离或上下文清理问题，另一种认为是模型幻觉把零散线索拼接成了看似“外来”的内容。由于这类问题一旦被证实，就不仅是体验问题，还会触及多租户环境中的安全边界，因此格外敏感。

**风险提醒：** 目前仍属社区报告，尚无官方定性，但对依赖 Agent 编程工作流的团队来说，这类事件需要持续关注。

### [社区报告 Codex 推理token异常聚类或致复杂任务表现下降](https://github.com/openai/codex/issues/30364) `#3`

> 据多名开发者反映，GPT-5.5 在 Codex 环境中的推理 token 数量出现异常聚类，频繁落在 516 及以 518 为步长的固定值上，导致部分复杂任务准确率下降。官方尚未回应，社区已提出临时规避方案。

多名开发者观察到 Codex 中 GPT-5.5 的推理 token 使用存在异常模式，数值频繁集中在 516、1034、1552 等固定档位。社区怀疑这可能意味着某种隐式截断、策略模板或推理预算机制在起作用，从而压缩了复杂任务所需的“思考空间”。如果这一判断成立，那么问题就不只是统计上的异常，而是会直接影响复杂编码、排障和多步骤任务的成功率。

**观察点：** 这条消息对重度使用 Codex 的用户更重要，短期内可能需要准备替代模型或临时规避提示词方案。

> Anthropic 近期宣布启动自主药物研发，重点攻克传统药企因回报有限而长期忽视的罕见病。但有专家提醒，AI 仍无法替代漫长实验与监管审批流程。

### [报道称 Midjourney 动议要求好莱坞片厂全面披露内部 AI 使用](https://techcrunch.com/2026/07/04/midjourney-wants-hollywood-studios-to-reveal-the-details-of-their-ai-usage/) `#5`

> 据报道，Midjourney 在与好莱坞制片厂的版权诉讼中提交动议，要求 Disney、Universal 和 Warner Bros. 全面披露其自身的生成式 AI 使用情况，并指对方可能也在内部利用未授权版权内容训练模型。

这起诉讼的有趣之处在于，Midjourney 不再只是被动应对“是否侵权”，而是试图把焦点拉向“行业内是否普遍存在类似做法”。如果法院允许更大范围的证据开示，好莱坞制片厂内部如何使用生成式 AI、是否在生产流程中依赖版权素材训练或辅助生成，可能被迫曝光。这会让版权争议从单一案件演变成更广泛的行业透明度议题。
