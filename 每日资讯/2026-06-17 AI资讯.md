## Codex 更新

### OpenAI Codex / Codex CLI

- **最新稳定版本**：`0.140.0` / `rust-v0.140.0`
- **发布时间**：2026-06-15 21:06 UTC（GitHub API）
- **预发布动态**：GitHub releases 页面显示 2026-06-16 有 `0.141.0-alpha.1`、`0.141.0-alpha.2`、`0.141.0-alpha.3` 预发布。

### 0.140.0 主要变化

- 新增 `/usage`：支持查看每日、每周、累计账号 token 活动。
- `/goal` 改进：可保留超大文本、大段粘贴内容和图片附件，包含 remote app-server sessions。
- 新增永久删除会话能力：`codex delete`、`/delete`、app-server thread/delete，并带确认保护与 subagent 清理。
- 新增 `/import`：可选择性从 Claude Code 导入 setup、项目配置和近期聊天。
- 输入 `@` 默认打开统一 mentions 菜单，覆盖文件、插件、skills。
- 新增 Amazon Bedrock API-key 管理认证，以及 CLI/MCP OAuth 凭据的加密本地存储。
- 修复 SQLite 状态库损坏自动备份/重建、MCP 启动/凭据、remote plugin uninstall、后台非 TTY 命令 Ctrl-C 等问题。
- 移除 TUI 中实验性的 `/realtime` voice controls 和相关音频依赖。

### 关注点

Codex 与 Claude Code 的互操作性增强值得关注：0.140.0 已出现从 Claude Code 导入配置/近期聊天的能力，这说明 coding agent 生态正在围绕迁移、兼容、会话资产复用展开竞争。

## Claude 更新

### Claude Code

- **最新版本**：`v2.1.179`
- **发布时间**：2026-06-16 20:22 UTC（GitHub API）

### 近期主要变化（v2.1.179 / v2.1.178 附近）

- 修复 mid-stream connection drops：保留部分响应，不再只显示原始错误，spinner 不再卡在 “running tool”。
- 修复 WSL2 下 Windows Terminal / VS Code 的鼠标滚轮滚动回归问题。
- 修复 Linux 大目录树 sandbox denyRead/allowRead glob 导致 Bash tool description 过大、会话不可用的问题。
- 修复 subagent transcript、prompt input focus、remote session background tasks 等体验问题。
- 新增 `Tool(param:value)` 权限规则语法，可按工具输入参数匹配，如 `Agent(model:opus)`。
- nested `.claude/skills` 目录的技能加载、名称冲突、closest `.claude/` 规则均有改进。
- auto mode 对 subagent 启动前做 classifier 检查，减少绕过 review 的风险。
- `/doctor`、Remote Control、`/bug`、Bedrock 凭据缓存、tmux/SSH 剪贴板、Windows background-service 等多处修复。

## CLIProxyAPI 更新

固定关注仓库：<https://github.com/router-for-me/CLIProxyAPI>

- **最新 release**：`v7.2.12`
- **发布时间**：2026-06-16 19:47 UTC（GitHub API）
- **最新 tag**：`v7.2.12`
- **对应 commit**：`8d2c00c107b2e62d0798bb2325224c178889e662`

### 最新提交摘要

提交信息：`feat(plugin-config): update default plugin Enabled behavior to false and expand test coverage`

主要内容：

- 将默认插件 `Enabled` 状态从 `true` 改为 `false`，涉及配置、运行时逻辑和 YAML 默认值。
- 新增 `enabledPluginConfigs` 辅助函数，用于生成显式启用插件配置。
- 扩展 `pluginhost`、`config`、`management` 单元测试，验证 disabled plugins、默认设置、跳过加载等场景。

### 影响判断

这是一个需要注意的行为变化：如果已有部署依赖“插件默认启用”，升级到 v7.2.12 后可能需要显式配置插件启用状态。建议升级前检查配置文件中的插件 `Enabled` 字段。
