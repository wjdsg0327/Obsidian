> Codex负责人Tibo宣布，临时取消Plus、{Business |"Business "}和 Pro 用户的5小时使用限额，并为庆祝活跃用户达到600万执行了一次即时用量重置，同时在网页端和移动端上线使用已储存重置机会的功能。团队还将优化GPT-5.6 Sol的用量效率，并在桌面端推出一批更新回应用户反馈。针对{特定两小时窗口|"特定两小时窗口"}内{部分用户|"部分用户"}重置未生效的问题，Tibo表示将向该时段所有点击过重置按钮的用户补发一次可储存的重置机会。他还宣布，次日将庆祝活跃用户达到700万，并首次向所有Codex用户发放一次可储存的重置机会。

Codex 负责人 Tibo 宣布针对 Codex 和 ChatGPT Work 的多项更新。官方已面向所有 Plus、Business 及 Pro 计划付费用户临时取消了 5 小时使用限额。为庆祝达到 600 万活跃用户，官方已在发布声明后一小时内执行了一次即时用量重置。

关于后续计划，Tibo 宣布将在明天庆祝 700 万活跃用户里程碑时，首次面向所有用户发放一次重置次数。

### Codex 负责人分享 Claude Code 调用 GPT 5.6 Sol 配置方案 `#4`
> Codex 负责人 Tibo 分享了一种在 Claude Code 中调用 GPT 5.6 Sol 模型的方法。据其介绍，用户可通过配置 CLIProxyAPI 并设置一个名为“claudex”的命令行别名，即可让 Claude Code 使用 OpenAI 的最新模型。然而有用户称，使用帖文提及的 CLIProxyAPI 后，自己的账号被停用了。

据 Codex 负责人 Tibo 分享，通过 CLIProxyAPI 工具和特定别名配置，已可在 Claude Code 中直接调用 GPT 5.6 Sol 模型。该方法被描述为一种快速集成方案，整个设置过程被指仅需约五分钟，且无需安装 Codex 独立应用。其核心步骤包括安装 CLIProxyAPI、连接 Claude 和 Codex 认证，以及在终端定义一个“claudex”别名来设定子 Agent 模型与工具并发等环境变量。
```
alias claudex='CLAUDE_CODE_SUBAGENT_MODEL=gpt-5.6-sol \
CLAUDE_CODE_ALWAYS_ENABLE_EFFORT=1 \
CLAUDE_CODE_MAX_TOOL_USE_CONCURRENCY=3 \
ENABLE_TOOL_SEARCH=false \
claude --model gpt-5.6-sol'
```

### [OpenAI 工作人员辟谣 GPT-5.6 Sol 上下文双倍计费传闻](https://x.com/pvncher/status/2076014465489817708) `#5`
> 近日，社区流传一种通过修改 Codex 配置文件来降低 GPT-5.6 系列模型 token消耗的方法，但 OpenAI Codex 官方工作人员辟谣，称该做法不正确并提醒用户谨慎修改。官方强调，GPT-5.6 系列模型在超过 272K tokens 的默认扩展上下文窗口后，并不会对订阅用户收取更高费用，价目表所示的倍率仅为 API 定价。

针对社区中用户关于 GPT-5.6 Sol 额度消耗过快的广泛讨论，OpenAI Codex 团队工作人员已出面辟谣，驳斥了广为流传的“超过 272K tokens 开始双倍计费”的说法，并呼吁用户保持默认配置。该争议源于社区用户发布的一个“省 token 教程”，建议在配置文件中手动限制上下文窗口，官方明确否定了这一操作的有效性与安全性，指出对订阅用户而言，基础价并未上涨，仅在订阅计划外手动扩展窗口才会触发额外计费。
