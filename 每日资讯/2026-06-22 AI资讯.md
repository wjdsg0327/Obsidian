## 重点新闻

### 1. OpenAI 提出/被报道新的部署前风险预测方法：Deployment Simulation

- BankInfoSecurity 报道 OpenAI 研究者提出一种名为 **Deployment Simulation** 的评估思路，用更接近真实生产环境的用户提示来降低模型“知道自己正在被测试”导致的评估偏差。
- 报道称该方法用真实近期 ChatGPT 用户对话样本（来自同意数据使用的用户）替换原响应，再比较候选模型响应，并用评估模型对常见不当行为进行分类预测。
- 需要注意：该方法依赖 OpenAI 内部真实用户数据，外部独立审计难度较高；报道也提到可用 WildChat 等公开数据集作为松散 proxy，但并不等价。

### 2. Arbor 自主优化框架被报道：同预算下优于 Claude Code / Codex

- VentureBeat 报道，来自中国人民大学与 Microsoft Research 的研究提出 **Arbor** 自主优化框架，将假设、实验、证据与洞察组织成树状结构，并用隔离 worktree 验证不同优化假设。
- 报道称 Arbor 在若干真实工程/机器学习优化任务中，取得超过 Claude Code 与 Codex 平均相对收益 2.5 倍的可验证提升；例如 BrowseComp 搜索 agent 任务中，held-out accuracy 从 45.33% 提升到 67.67%。
- 解读：这类框架体现了“loop engineering / 长周期 agent 工程”的方向——不是让单个 agent 更久地循环，而是让尝试、失败、证据、merge gate 可追踪、可复盘。实际采用时要关注 evaluator 是否可靠、token/算力成本、并发 worktree 的工程复杂度。

### 3. AI Agent 商业化与支付能力继续推进

- 搜索结果显示，近期围绕 **AI agents 执行交易/支付、企业采用、代理式工作流** 的报道较多。例如 OpenTools 聚合页提到 Visa 与 OpenAI 平台支付能力合作相关消息；该类消息需以后续官方公告核对。
- 对老王的实用意义：后续如果 AI agent 可直接触发支付、采购、订阅等外部动作，权限边界、审批流、限额与审计会变得更重要。

## Codex 更新

### OpenAI Codex CLI / Codex GitHub Release

- **最新版本**：`rust-v0.141.0`（Release 名称：`0.141.0`）
- **发布时间**：2026-06-18 04:43 UTC
- **来源**：GitHub Release：<https://github.com/openai/codex/releases/tag/rust-v0.141.0>

### 主要变化

- **远程执行安全增强**：remote executors 使用经过认证的端到端加密 **Noise relay channels**。
- **跨平台远程执行改进**：远程执行现在能更好保留 executor 原生工作目录与 shell，包括 app-server 与 exec-server 边界上的文件系统权限路径。
- **插件 / MCP 能力增强**：选定 executor plugins 可按 thread 激活 stdio MCP servers；插件发现新增 “created-by-me marketplace” 与按认证方式区分的 curated catalogs。
- **App-server 能力**：客户端可列出 immediate child threads，关联 external-agent imports 与详细结果，并读取/兑换 rate-limit reset credits。
- **Realtime/TUI 改进**：Realtime clients 可显式追加 speech、控制 Codex 响应进入对话方式、忽略 startup context；TUI 输入提示可在空闲后自动解决，并带可暂停倒计时。
- **修复**：Windows sandbox stale credentials 自动修复；PowerShell 命令后台化前等待更久；SQLite 固定到包含 WAL-reset corruption fix 的版本；TLS 支持企业代理常见的 P-521 证书签名。

### 对使用者的影响

- 如果老王使用 Codex 做远程节点执行或跨平台执行，`0.141.0` 的路径/工作目录保留与远程 relay 改动值得关注。
- 如果使用插件/MCP，需留意认证模式下插件 capability 路由与 MCP 去重逻辑的变化。

## OpenClaw 更新

### GitHub Release

- **仓库**：<https://github.com/openclaw/openclaw>
- **最新版本**：`v2026.6.9`（Release 名称：`openclaw 2026.6.9`）
- **发布时间**：2026-06-21 01:44 UTC
- **Release 链接**：<https://github.com/openclaw/openclaw/releases/tag/v2026.6.9>

### 重点变化

- **Telegram 投递增强**：支持更丰富 HTML、保留 rich markdown 与 sticker 路径，更好渲染 progress drafts 与 command output，安全规范化 HTML tables，并改进 mention / spooled handler 投递路径。
- **Agent / session runtime 可靠性增强**：更可靠的 retries、terminal outcomes、compaction 后 usage 保留、session history repair、reply reconciliation，提升中断/部分失败 turn 的恢复能力。
- **Codex 集成增强**：Codex 获得 automatic plugin approvals、GPT-5.3 Spark OAuth routing、remote-node `exec` 动态工具，以及更可靠的 app-server teardown / terminal outcomes。
- **官方 provider plugins 外部化**：外部 provider packages 成为一等 npm releases；Gateway 启动时可加载外部安装的 channel plugins；StepFun 可从 npm 与 ClawHub 获取。
- **Web / Native 客户端增强**：Control UI 增加 session workspace rail 与 extension health；iOS 增加 Watch controls；Android 显示 chat context。
- **搜索与技能**：Codex Hosted Search 可用；key-free search providers 保持显式 opt-in；ClawHub skill installs 保留 verified source provenance。

### 对当前使用的影响

- 这次 OpenClaw 更新与“每日资讯任务”相关的重点是：cron 列表更紧凑、agent 恢复更稳、Codex 集成更强、插件来源可追溯性提升。
- 如需升级，建议先查看完整 Release notes 与本机配置差异，避免 provider/channel 插件变更影响现有 QQBot、cron、知识库写入流程。

## Hermes Agent 更新

### GitHub Release

- **仓库**：<https://github.com/NousResearch/hermes-agent>
- **最新版本**：`v2026.6.19`（Release 名称：`Hermes Agent v0.17.0`）
- **发布时间**：2026-06-19 19:39 UTC
- **Release 链接**：<https://github.com/NousResearch/hermes-agent/releases/tag/v2026.6.19>

### 重点变化：v0.17.0 “Reach Release”

- **iMessage 支持**：通过 Photon Spectrum 的 iMessage platform plugin，让 Hermes 可收发 iMessage；Release notes 称无需 Mac relay / BlueBubbles bridge。
- **Raft agent network**：新增 Raft platform adapter，可作为 external agent 通过 wake-channel bridge 接入 Raft；设计上 wake payload 只携带 metadata，不携带消息正文。
- **桌面端增强**：可重绑快捷键、原生系统通知、subagent watch-windows、composer model selector、RTL/bidi 自动方向、VS Code 风格 terminal pane、per-thread drafts、安装 VS Code Marketplace theme 等。
- **后台/异步 subagents**：`delegate_task(background=true)` 可让 subagent 后台运行并立即返回 handle，完成后结果作为新 turn 回到对话。
- **图像编辑**：`image_generate` 支持基于源图的编辑/转换，不仅是从零生成。
- **Automation Blueprints**：让用户通过表单/对话式参数配置自动化任务，减少直接写 cron 语法。
- **模型/工具生态**：Release notes 提到 Cursor Composer model 可通过 xAI Grok subscription 访问；dashboard profile builder、Skills Hub browser、`memory` tool 也有升级。

### 对使用者的影响

- Hermes v0.17.0 明显朝“多渠道个人 agent + 团队 dashboard + 后台委托任务”方向推进。
- 若老王关注 iMessage/桌面端/后台 subagent，这个版本值得单独研究；生产环境仍建议按 tag 固定并先在测试环境验证。
