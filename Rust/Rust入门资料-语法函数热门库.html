Rust 入门资料：语法、函数与热门库

适合已经会一点编程、想系统入门 Rust 的学习路线。重点不是背语法，而是理解 Rust 的核心：所有权 + 类型系统 + 零成本抽象。

建议节奏：先跑通环境 → 写小程序 → 重点攻克所有权 → 再看生态库和项目。

学习路线

Rust 的学习曲线比 Python、JavaScript 陡一些，但它的难点很集中：一开始会被所有权、借用、生命周期卡住；一旦过了这关，后面的工程体验会非常稳。

Rust 学习路径

推荐顺序

  1. 先安装 Rust，熟悉 cargo new、cargo run、cargo test。
  2. 学基础语法：变量、类型、控制流、函数。
  3. 集中攻克所有权、借用、引用、生命周期。
  4. 学结构体、枚举、模式匹配、错误处理。
  5. 再学模块、测试、常用库，最后做小项目。
常见误区
不要一开始就死磕生命周期标注。先会写简单函数和结构体，再看编译器报错，它会一步步教你。

环境与 Cargo

Rust 官方工具链叫 rustup，包管理和构建工具叫 cargo。实际开发里，Cargo 是每天都要用的入口。

Cargo 项目结构
# 安装后常用命令 rustc --version cargo new hello_rust cd hello_rust cargo run cargo build --release cargo test cargo fmt cargo clippy

Cargo.toml 示例

[package] name = "hello_rust" version = "0.1.0" edition = "2021" [dependencies] serde = { version = "1", features = ["derive"] }

把 Cargo 理解为 Rust 的“项目管家”：下载依赖、编译、测试、格式化、发布都靠它。

常见误区
不要直接用 rustc main.rs 管大项目。学习单文件可以，正式项目用 Cargo。

基础语法

Rust 默认变量不可变，这是它鼓励安全代码的第一步。需要修改时显式写 mut。

变量绑定与可变性

常见类型

  • 整数：i32、i64、u32、usize
  • 浮点：f32、f64
  • 布尔：bool
  • 字符：char，是 Unicode 标量值，不只是 ASCII
  • 字符串：String 和 &str
  • 复合类型：元组、数组、切片、结构体、枚举
fn main() { let x = 10; // 不可变 let mut y = 20; // 可变 y += 1; let name: &str = "laowang"; let title: String = String::from("Rust 入门"); let point = (10, 20); let nums = [1, 2, 3, 4]; println!("x={x}, y={y}, name={name}, title={title}"); println!("point=({}, {}), first={}", point.0, point.1, nums[0]); }

控制流

let score = 82; let level = if score >= 90 { "优秀" } else if score >= 60 { "及格" } else { "继续练" }; for i in 0..5 { println!("{i}"); } let mut n = 3; while n > 0 { n -= 1; }
常见误区
Rust 的 if 可以作为表达式返回值，但每个分支返回类型必须一致。

函数与表达式

Rust 函数用 fn 定义，参数必须写类型，返回值用 ->。函数体最后一行如果没有分号，就是返回表达式。

函数：输入 → 处理 → 输出
fn add(a: i32, b: i32) -> i32 { a + b // 没有分号，作为返回值 } fn print_msg(msg: &str) { println!("{msg}"); // 无返回值，相当于返回 () } fn main() { let result = add(3, 5); print_msg("hello rust"); println!("result = {result}"); }

函数里的表达式

let x = { let a = 10; let b = 20; a + b }; // x = 30

Rust 更偏表达式风格。很多结构都能产出值，例如 if、代码块、match。

常见误区
a + b; 有分号就变成语句，返回 ()，这会导致返回类型不匹配。

所有权、借用与生命周期

这是 Rust 最核心也最容易卡住的部分。简单说：每个值只有一个所有者；所有者离开作用域，值就被释放；想临时使用别人的值，用引用借用。

所有权移动与借用

所有权移动

let s1 = String::from("hello"); let s2 = s1; // s1 的所有权移动给 s2 // println!("{s1}"); // 这里会报错：s1 已经不能用了 println!("{s2}");

借用引用

fn len_of(s: &String) -> usize { s.len() } fn main() { let s = String::from("hello"); let len = len_of(&s); // 借用，不拿走所有权 println!("{s}, len={len}"); }

可变借用

fn push_world(s: &mut String) { s.push_str(" world"); } let mut s = String::from("hello"); push_world(&mut s);

规则可以先记成两条：同一时间可以有多个不可变引用；或者一个可变引用；两者不能同时存在。

常见误区
生命周期不是“让变量活得更久”的魔法。生命周期标注只是告诉编译器：返回引用和输入引用之间是什么关系。

结构体、枚举与模式匹配

结构体适合表达“一个东西有很多字段”，枚举适合表达“一个值可能是几种状态之一”。Rust 的枚举很强，可以携带数据。

结构体 vs 枚举

结构体

struct User { name: String, age: u8, } impl User { fn new(name: String, age: u8) -> Self { Self { name, age } } fn say_hi(&self) { println!("我是 {}，{} 岁", self.name, self.age); } }

枚举和 match

enum Direction { Up, Down, Left, Right, } fn move_to(d: Direction) { match d { Direction::Up => println!("向上"), Direction::Down => println!("向下"), Direction::Left => println!("向左"), Direction::Right => println!("向右"), } }

match 必须覆盖所有情况，这让很多遗漏在编译期就被抓出来。

常见误区
不要把 Rust 的 enum 当成 C 语言那种纯数字枚举。Rust 的 enum 可以携带不同类型的数据，非常适合建模状态。

错误处理、泛型与 Trait

Rust 不鼓励到处抛异常。可恢复错误常用 Result<T, E>，可能为空常用 Option<T>。

Result 错误流

Result 与 ?

use std::fs; use std::io; fn read_config() -> Result { let text = fs::read_to_string("config.toml")?; Ok(text) }

? 的意思是：如果是 Ok 就取出值继续；如果是 Err 就提前返回错误。

泛型与 Trait

trait Summary { fn summary(&self) -> String; } struct Article { title: String, } impl Summary for Article { fn summary(&self) -> String { format!("文章：{}", self.title) } } fn print_summary(item: &T) { println!("{}", item.summary()); }

Trait 可以理解为“能力接口”：一个类型实现了某个 Trait，就说明它具备某种能力。

常见误区
不要一上来就写复杂泛型。先把具体类型代码写清楚，再抽象成 Trait。

集合、迭代器、闭包、模块与测试

实际写 Rust，常见容器是 Vec、HashMap、String。迭代器和闭包让数据处理很舒服。

数据处理管道

集合与迭代器

let nums = vec![1, 2, 3, 4, 5]; let doubled_even: Vec = nums .iter() .filter(|n| **n % 2 == 0) .map(|n| n * 2) .collect(); println!("{:?}", doubled_even); // [4, 8]

模块

// src/lib.rs pub mod user; // src/user.rs pub fn create_user(name: &str) { println!("create {name}"); }

测试

fn add(a: i32, b: i32) -> i32 { a + b } #[cfg(test)] mod tests { use super::*; #[test] fn test_add() { assert_eq!(add(2, 3), 5); } }
常见误区
迭代器是惰性的。只写 map 不一定执行，通常需要 collect、sum、for_each 等消费它。

并发与异步

Rust 的并发优势在于：很多线程安全问题会在编译期被阻止。普通并发可以用线程和 channel；网络服务常用 async 生态，比如 Tokio。

并发模型：线程与消息

线程示例

use std::thread; fn main() { let handle = thread::spawn(|| { for i in 1..=3 { println!("子线程 {i}"); } }); handle.join().unwrap(); }

Tokio 异步示例

#[tokio::main] async fn main() { let body = reqwest::get("https://www.rust-lang.org") .await .unwrap() .text() .await .unwrap(); println!("len = {}", body.len()); }

异步代码里的 await 不是阻塞线程，而是把当前任务让出去，等 IO 准备好后再继续。

常见误区
不要把 async 当成“自动变快”。它适合大量 IO 等待，比如网络请求、数据库、文件服务；CPU 密集任务还是要考虑线程池。

热门库与练习项目

Rust 生态非常偏工程化。入门时不需要把库全学完，按项目需求挑即可。

Rust 生态地图

热门库索引

serde
序列化/反序列化，JSON、YAML、TOML 都常用。几乎是 Rust 数据交换标配。
tokio
异步运行时。写高并发网络服务、定时任务、异步 IO 常用。
reqwest
HTTP 客户端。调用 API、下载网页、发请求很方便。
axum
Web 框架，基于 tokio/tower，适合写 REST API。
actix-web
成熟高性能 Web 框架，生态久，资料多。
clap
命令行参数解析。做 CLI 工具优先看它。
anyhow
应用层错误处理，适合快速开发 CLI/服务。
thiserror
库层自定义错误类型，适合写更规范的错误。
tracing
结构化日志和链路追踪，比 println 更适合服务端。
sqlx
异步 SQL 数据库访问，支持 PostgreSQL、MySQL、SQLite。
diesel
类型安全 ORM，偏传统 ORM 风格。
rayon
数据并行库，CPU 密集任务并行处理非常方便。
chrono / time
日期时间处理。新项目也可以看看 time。
regex
正则表达式库，稳定常用。
bevy
游戏引擎，适合对 ECS 和游戏开发感兴趣时学习。

推荐练习项目

  1. CLI Todo：用 clap 解析命令，用 serde 保存 JSON 文件。
  2. 日志分析器：读取大文件，统计错误行、IP、接口耗时。
  3. HTTP 小工具：用 reqwest 请求 API，格式化输出结果。
  4. 短链接服务：用 axum + sqlx + PostgreSQL 做一个 Web API。
  5. 并发下载器：用 tokio 同时下载多个 URL，练 async。
// 一个简单 CLI 依赖组合 [dependencies] clap = { version = "4", features = ["derive"] } serde = { version = "1", features = ["derive"] } serde_json = "1" anyhow = "1"
常见误区
别从大型 Web 项目开局。Rust 新手最适合先写 CLI 和小工具，因为反馈快、依赖少、能集中练语法和所有权。
