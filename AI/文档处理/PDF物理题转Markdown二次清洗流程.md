# PDF 物理题转 Markdown：二次清洗流程

> 整理时间：2026-06-08  
> 适用场景：物理试卷、讲义、答案解析、含公式 PDF 转 Markdown。  
> 核心结论：**MinerU / marker 只负责抽取初稿，公式必须二次清洗，不能完全指望 PDF 转换工具自动做好。**

---

## 1. 总体流程

```text
PDF / 扫描件 / 试卷 / 答案解析
        ↓
MinerU / marker 抽取文本、图片、表格、基础 Markdown
        ↓
脚本规则 + 人工校对二次清洗物理公式
        ↓
按题号重排 Markdown
        ↓
进入知识库 / 题库 / RAG / AI 批改系统
```

推荐流程：

1. **先抽取**：用 MinerU 或 marker 把 PDF 转成 Markdown / JSON / 图片资源。
2. **再清洗公式**：用脚本规则或人工规则把常见物理公式重写成 LaTeX。
3. **再按题号整理**：统一题号、题干、选项、答案、解析、分值等结构。
4. **最后人工抽查**：重点检查公式、单位、指数、上下标、分式、希腊字母。

---

## 2. 为什么不能完全依赖 PDF 转换工具

PDF 转换工具容易在物理题里出错，尤其是：

- 分式识别错误：`v=s/t`、`p=F/S`、`I=U/R` 可能被抽成普通文本。
- 上下标丢失：`I²Rt`、`10^3`、`m^3`、`N m^-2` 容易变形。
- 希腊字母错误：`ρ`、`η`、`μ`、`φ`、`Δ` 可能变成乱码或相似字符。
- 单位错误：`kg/m3`、`N/m2`、`m s-1`、`kg m-3` 需要规范化。
- 表格和多栏题顺序混乱。
- 页眉、页脚、页码、水印混入正文。

所以更可靠的定位是：

> **MinerU / marker = 初稿抽取工具；公式清洗 = 单独处理环节。**

## 3. 公式二次清洗原则

### 3.1 基本原则

- 所有公式统一改写为 LaTeX。
- 行内公式使用 `$...$`。
- 独立公式使用：

```markdown
$$
...
$$
```

- 明显公式不要保留 OCR 乱码。
- 无法确定的公式标记为：`【公式待核对】`。
- 不要为了“看起来完整”而瞎补公式。

### 3.2 常见物理公式映射

| 原始/OCR 常见形式 | 建议 LaTeX |
|---|---|
| `v = s / t` | `$v = \frac{s}{t}$` |
| `a = Δv / Δt` | `$a = \frac{\Delta v}{\Delta t}$` |
| `F = ma` | `$F = ma$` |
| `G = mg` | `$G = mg$` |
| `p = F / S` | `$p = \frac{F}{S}$` |
| `W = Fs` | `$W = Fs$` |
| `P = W / t` | `$P = \frac{W}{t}$` |
| `η = W有 / W总 × 100%` | `$\eta = \frac{W_{\text{有}}}{W_{\text{总}}} \times 100\%$` |
| `Q = cmΔt` | `$Q = cm\Delta t$` |
| `ρ = m / V` | `$\rho = \frac{m}{V}$` |
| `I = U / R` | `$I = \frac{U}{R}$` |
| `P = UI` | `$P = UI$` |
| `Q = I2Rt` / `Q = I²Rt` | `$Q = I^2Rt$` |
| `E = Pt` | `$E = Pt$` |
| `F = BIL` | `$F = BIL$` |
| `Φ = BA` / `phi = BA` | `$\Phi = BA$` 或 `$\phi = BA$` |
| `Δ(mv)=FΔt` | `$\Delta(mv) = F\Delta t$` |

### 3.3 常见 OCR 符号修正

| 错误/不统一形式 | 规范形式 |
|---|---|
| `△` | `\Delta` |
| `Δ` | `\Delta` |
| `×10 的 n 次方` | `\times 10^{n}` |
| `x10^n` | `\times 10^{n}` |
| `I2Rt` | `I^2Rt` |
| `kg/m3` | `kg/m^3` |
| `N/m2` | `N/m^2` |
| `m/s` | `m\,s^{-1}` 或 `\mathrm{m/s}` |
| `ms-1` | `m\,s^{-1}` |
| `kgm-3` | `kg\,m^{-3}` |

## 4. Markdown 题目整理格式

建议统一为：

```markdown
## 第 1 题

**题目：**
题干内容……

**选项：**
A. ...
B. ...
C. ...
D. ...

**答案：** A

**解析：**
解析内容……

**公式：**
$$
...
$$

**分值：** 1
```

如果是英文原卷，也可以保留：

```markdown
## Question 1

**Answer:** A

**Explanation:**
...

**Mark:** 1
```

原则：

- 每道题单独成块。
- 题号层级统一用二级标题 `##`。
- 答案、解析、分值字段统一。
- 原文图片保留引用位置，如：`![图1](images/图1.png)`。
- 不确定内容用 `【待核对】` 标注。

## 5. 可让 AI 执行的提示词

```text
我有一批物理题 PDF，需要整理成高质量 Markdown 笔记。

请按下面流程处理，不要完全依赖 PDF 转换工具的公式识别结果：

1. 先使用 MinerU 或 marker 从 PDF 中抽取文本和基础 Markdown。
2. 对抽取结果进行二次清洗：
   - 检查常见物理公式是否识别错误。
   - 将公式统一重写为标准 LaTeX。
   - 不要保留乱码公式、半截公式、错误 OCR 符号。
   - 对明显的物理表达式进行人工规则修正。
3. 按题号整理 Markdown：
   - 每道题单独成块。
   - 保留题干、选项、解析、答案。
   - 图片、图表保留引用位置。
4. 输出要求：
   - 最终结果必须是干净 Markdown。
   - 所有公式用 LaTeX。
   - 行内公式用 `$...$`。
   - 独立公式用 `$$...$$`。
   - 删除页眉、页脚、页码、水印。
   - 无法确定的公式标记为 `【公式待核对】`。

请先给出处理脚本或处理流程，然后对我提供的 PDF / Markdown 进行清洗整理。
```

## 6. 脚本化处理建议

可以写一个 Python 清洗脚本，分成几层：

1. **输入层**：读取 MinerU / marker 生成的 Markdown。
2. **结构层**：按题号切分，如 `1.`、`1、`、`第1题`、`Question 1`。
3. **清理层**：删除页眉页脚、页码、水印、重复标题。
4. **公式层**：用规则表替换常见物理公式。
5. **格式层**：输出统一 Markdown 模板。
6. **人工核对层**：保留 `【公式待核对】` 标记，方便后续检查。

公式规则应该单独放在字典或配置文件里，便于不断扩展。
