# 01-核心概念：场景、节点、组件、脚本

## Cocos 的核心模型

学 Cocos Creator，最重要的是先建立一个脑内模型：

```text
Scene 场景
└─ Node 节点
   ├─ Transform 位置/旋转/缩放
   ├─ Component 组件
   └─ Child Nodes 子节点
```

你看到的任何东西，基本都挂在某个节点上。

## 场景 Scene

场景就是一个游戏画面或关卡。

例子：

- 主菜单场景
- 第一关场景
- 战斗场景
- 结算场景

初学可以先一个项目只做一个场景。

## 节点 Node

节点是 Cocos 里最基础的对象。节点本身不一定能显示东西，它更像一个“挂载点”。

节点常见属性：

- Position：位置
- Rotation：旋转
- Scale：缩放
- Active：是否启用
- Parent/Children：父子关系

一个角色可能长这样：

```text
Player(Node)
├─ Sprite 组件：显示图片
├─ Collider 组件：参与碰撞
└─ PlayerController 脚本：控制移动
```

## 组件 Component

组件负责给节点添加能力。

常见组件：

- Sprite：显示图片
- Label：显示文字
- Button：按钮
- AudioSource：播放声音
- Collider2D：碰撞检测
- RigidBody2D：物理刚体

Cocos 的思想是：

> 节点负责“是谁”和“在哪里”，组件负责“能做什么”。

## 脚本 Script

你写的 TypeScript 脚本，本质上也是组件。

例如：

```ts
import { _decorator, Component } from 'cc';
const { ccclass } = _decorator;

@ccclass('PlayerController')
export class PlayerController extends Component {
  start() {
    console.log('玩家启动');
  }
}
```

把这个脚本拖到 Player 节点上，Player 就拥有了 `PlayerController` 这个能力。

## 生命周期

脚本组件有几个常见生命周期：

```text
onLoad   -> 节点加载时，只执行一次
start    -> 第一次 update 前执行一次
update   -> 每一帧执行
onDestroy-> 节点销毁时执行
```

最常用：

- 初始化：写在 `onLoad` 或 `start`
- 持续移动：写在 `update`

## 一个最小例子

让节点每秒向右移动：

```ts
import { _decorator, Component, Vec3 } from 'cc';
const { ccclass } = _decorator;

@ccclass('MoveRight')
export class MoveRight extends Component {
  speed = 100;

  update(deltaTime: number) {
    const pos = this.node.position.clone();
    pos.x += this.speed * deltaTime;
    this.node.setPosition(pos);
  }
}
```

这里的关键点：

- `this.node` 是脚本挂载的节点
- `deltaTime` 是上一帧到这一帧的时间
- 用 `speed * deltaTime` 能保证不同电脑速度相对一致

## 常见误区

### 误区 1：以为脚本是独立运行的

脚本要挂到节点上才会发挥作用。

### 误区 2：所有代码都写一个大脚本

入门可以先这样，但后面要拆：

- PlayerController 控制玩家
- Coin 控制金币
- GameManager 管理分数和状态

### 误区 3：不理解父子节点

子节点的位置是相对父节点的。父节点移动，子节点会跟着动。
