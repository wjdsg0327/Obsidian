# 09-发布到 Web 和小游戏平台

## 发布前先分清两个动作

- **预览 Preview**：开发时快速看效果
- **构建 Build**：生成可发布文件

不要把预览地址当成正式发布。

## 发布 Web/H5

大致流程：

1. 菜单打开构建发布面板
2. 平台选择 Web Desktop 或 Web Mobile
3. 设置构建目录
4. 点击 Build
5. 点击 Make/Run 或用本地服务器测试

构建后会生成一堆文件，通常需要放到 Web 服务器上。

## 为什么不能直接双击 index.html

很多 Web 游戏资源加载依赖 HTTP 服务。直接双击打开是 `file://`，可能导致资源加载失败。

本地可以用简单服务器：

```bash
python -m http.server 8080
```

然后浏览器访问：

```text
http://localhost:8080
```

## 微信小游戏发布思路

大致流程：

1. Cocos 构建平台选择微信小游戏
2. 填 AppID 或测试号
3. 构建生成小游戏项目
4. 用微信开发者工具打开
5. 预览、真机调试、上传

注意：

- 微信小游戏有包体限制
- 资源可能需要放 CDN
- API 和浏览器不同
- 要用微信开发者工具调试

## 构建常见设置

你先重点关注：

- 起始场景是否正确
- 设计分辨率是否合理
- 横屏/竖屏
- 资源是否都在 assets 下
- 有没有红色编译错误

## 适配屏幕

2D 小游戏常见问题：不同手机尺寸不一样。

入门建议：

- 先固定竖屏
- Canvas 设置合适设计分辨率，例如 720x1280
- UI 用 Widget 组件贴边
- 不要把重要按钮放太靠边

## 发布前检查清单

- [ ] 没有 Console 红色报错
- [ ] 第一场景正确
- [ ] 按钮都能点
- [ ] 游戏结束能重开
- [ ] 手机比例下 UI 不乱
- [ ] 音效不刺耳
- [ ] 资源大小可接受

## 常见错误

### 构建成功但打开黑屏

检查起始场景、Console 报错、资源路径。

### 编辑器能跑，微信工具不能跑

可能用了浏览器特有 API，或者资源/包体限制。

### UI 在手机上错位

使用 Widget、Layout，减少绝对坐标硬摆。
