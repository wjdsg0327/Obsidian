# 05-UI 系统：按钮、文本、进度条

## UI 也是节点

Cocos 的 UI 不是什么特殊世界，它也是节点加组件。

常见 UI：

- Label：文字
- Button：按钮
- Sprite：图片
- ProgressBar：进度条
- Layout：布局

一般放在 Canvas 下。

## Label 显示分数

脚本引用 Label：

```ts
import { _decorator, Component, Label } from 'cc';
const { ccclass, property } = _decorator;

@ccclass('ScoreUI')
export class ScoreUI extends Component {
  @property(Label)
  scoreLabel: Label | null = null;

  score = 0;

  start() {
    this.refresh();
  }

  addScore(value: number) {
    this.score += value;
    this.refresh();
  }

  refresh() {
    if (this.scoreLabel) {
      this.scoreLabel.string = `分数：${this.score}`;
    }
  }
}
```

## Button 点击事件

方式一：编辑器里绑定

1. 选中 Button 节点
2. 找到 Button 组件
3. Click Events 添加一项
4. 拖入带脚本的节点
5. 选择脚本方法

方法需要是 public：

```ts
public onStartButtonClick() {
  console.log('开始游戏');
}
```

方式二：代码绑定，入门先用编辑器绑定更直观。

## ProgressBar 血条/进度条

```ts
import { _decorator, Component, ProgressBar } from 'cc';
const { ccclass, property } = _decorator;

@ccclass('LifeBar')
export class LifeBar extends Component {
  @property(ProgressBar)
  bar: ProgressBar | null = null;

  setLife(current: number, max: number) {
    if (!this.bar) return;
    this.bar.progress = current / max;
  }
}
```

`progress` 范围是 0 到 1。

## UI 层级建议

```text
Canvas
└─ UI
   ├─ TopBar
   │  ├─ ScoreLabel
   │  └─ LifeLabel
   ├─ StartPanel
   │  └─ StartButton
   └─ GameOverPanel
      └─ RestartButton
```

通过 `node.active` 控制显示隐藏：

```ts
this.gameOverPanel.active = true;
```

## 常见误区

### UI 和游戏对象混在一起

建议 UI 放 UI 节点下，游戏对象放 GameRoot 或各自容器下。

### 直接用 console 当 UI

调试可以，真正游戏状态要显示在 Label 上。

### 按钮点了没反应

检查：

1. Button 组件是否在节点上
2. Click Events 是否绑定了正确节点
3. 方法名是否选择正确
4. 运行时 Console 是否报错
