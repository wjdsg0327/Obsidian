# 02-编辑器入门：界面、资源和场景

## 编辑器不是“画图工具”

Cocos Creator 编辑器的作用是：

- 摆放节点
- 配置组件
- 管理资源
- 连接脚本变量
- 预览运行结果

真正的游戏逻辑，还是靠脚本完成。

## Hierarchy：层级管理器

Hierarchy 显示当前场景里的节点树。

典型 2D 游戏结构：

```text
Canvas
├─ Background
├─ Player
├─ CoinContainer
├─ UI
│  ├─ ScoreLabel
│  ├─ LifeLabel
│  └─ StartButton
└─ GameManager
```

建议：

- UI 节点统一放到 `UI` 下
- 动态生成的金币放到 `CoinContainer` 下
- 管理逻辑放到 `GameManager` 空节点上

## Inspector：属性检查器

选中节点后，右边 Inspector 可以改：

- 节点名称
- 坐标、旋转、缩放
- 组件参数
- 脚本暴露出来的属性

脚本里这样写，可以在 Inspector 暴露字段：

```ts
import { _decorator, Component, Node } from 'cc';
const { ccclass, property } = _decorator;

@ccclass('Demo')
export class Demo extends Component {
  @property(Node)
  player: Node | null = null;
}
```

然后你就能在编辑器里把 Player 节点拖进去。

## Assets：资源管理器

`assets` 是你最常用的目录。

建议结构：

```text
assets/
├─ scenes/       # 场景
├─ scripts/      # 脚本
├─ textures/     # 图片
├─ prefabs/      # 预制体
├─ audio/        # 音频
└─ animations/   # 动画
```

不要所有东西都堆在根目录，后面会很乱。

## Scene 视图和 Game 视图

- Scene：编辑用，不一定等于玩家看到的画面
- Game：运行时玩家看到的画面

如果 Scene 里看到了，Game 里看不到，通常检查：

1. 节点是否在 Canvas 下
2. 摄像机是否能看到
3. 节点 Active 是否关闭
4. 层级或坐标是否跑到屏幕外

## 坐标系

2D 项目里你先记住：

- x 越大越靠右
- y 越大越靠上
- Canvas 中心通常是 `(0, 0)`

例如：

```text
        y+
        ↑
        |
x- ←----0----→ x+
        |
        ↓
        y-
```

## 预制体 Prefab

预制体是“可重复使用的节点模板”。

比如金币 Coin：

- 有 Sprite
- 有 Collider
- 有 Coin 脚本

你把 Coin 做成 Prefab 后，就可以用代码反复生成很多金币。

## 本章练习

整理项目目录：

```text
assets/scenes
assets/scripts
assets/textures
assets/prefabs
```

然后新建一个场景保存为：

```text
assets/scenes/Main.scene
```
