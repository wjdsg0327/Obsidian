# 11-进阶路线图

## 入门后先补这些能力

完成接金币小游戏后，下一步不要立刻做大项目。建议按下面顺序补能力。

## 1. 更熟练的 TypeScript

重点：

- class
- interface
- enum
- 类型声明
- 模块 import/export
- 数组和对象

小游戏项目不需要复杂 TS，但类型写清楚会少很多 bug。

## 2. Cocos 组件化设计

学会拆脚本：

```text
PlayerController：玩家输入和移动
Coin：金币自己的行为
GameManager：游戏状态
UIManager：界面刷新
AudioManager：音效播放
```

原则：

> 一个脚本只管一类事情。

## 3. 资源和预制体工作流

重点：

- Prefab 变体和复用
- 动态生成对象
- 对象池 Object Pool
- 资源加载和释放

对象池很适合子弹、金币、敌人这种频繁创建销毁的对象。

## 4. 动画系统

学习：

- Animation Clip
- 状态切换
- tween 动画
- UI 动效
- 角色帧动画

小游戏里动画很重要，能显著提升手感。

## 5. 音效和反馈

一个游戏好不好玩，很大程度来自反馈：

- 点击按钮有声音
- 接金币有音效
- 扣血有震动/闪烁
- 分数增加有动画

先做小反馈，不要一上来做复杂系统。

## 6. 数据保存

常见需求：

- 最高分
- 设置项
- 关卡进度

可以学习本地存储，例如 localStorage 或 Cocos 对应存储方案。

## 7. 平台能力

如果做微信小游戏，继续学：

- 微信登录
- 分享
- 排行榜
- 广告
- 分包
- CDN 资源

这些不是 Cocos 核心，但是真正上线会遇到。

## 8. 项目结构

建议结构：

```text
assets/
├─ scenes/
├─ scripts/
│  ├─ core/
│  ├─ gameplay/
│  ├─ ui/
│  └─ utils/
├─ prefabs/
├─ textures/
├─ audio/
└─ animations/
```

## 练手项目推荐

按难度排序：

1. 接金币
2. 打地鼠
3. Flappy Bird 简化版
4. 消消乐简化版
5. 塔防最小原型
6. 卡牌对战最小原型

每个项目只追求“核心玩法跑通”，不要一开始就做登录、商城、背包。

## 快速掌握标准

你能做到下面这些，就算真正入门：

- 能独立创建项目和场景
- 能用节点搭出画面
- 能写脚本控制节点
- 能用 UI 显示游戏状态
- 能用 Prefab 生成对象
- 能处理输入和碰撞
- 能构建 Web 版本
- 遇到报错知道先看 Console

## 后续学习建议

每天 1 小时，7 天安排：

```text
第 1 天：编辑器 + 节点 + Label/Button
第 2 天：脚本 + update + 移动
第 3 天：输入 + UI 刷新
第 4 天：Prefab + 生成金币
第 5 天：碰撞/距离判断 + 得分
第 6 天：游戏结束 + 重开 + 音效
第 7 天：构建 Web + 总结重构
```

这比连续看十小时视频更有效。
