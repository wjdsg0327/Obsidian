> 目标：不是把官方文档搬一遍，而是让你用最短路径理解 Cocos Creator 的工作方式，并能独立做出一个小型 2D 游戏原型。

## 适合谁

- 以前没系统学过游戏引擎
- 会一点点编程更好，不会也可以从这里开始
- 想做微信小游戏、H5 小游戏、2D 休闲游戏
- 想快速理解 Cocos Creator 的“场景、节点、组件、脚本、资源、发布”这一套

## 建议学习方式

每一章都按这个节奏来：

1. **先理解概念**：知道它解决什么问题
2. **照着做一遍**：不要只看
3. **改一个小地方**：比如速度、颜色、图片、按钮文字
4. **记录一个问题**：不会的地方留到下一轮查

## 你要形成的核心直觉

Cocos Creator 的开发方式可以概括成一句话：

> 在编辑器里搭场景，用节点组织对象，用组件赋予能力，用脚本写规则。

也就是：

```text
游戏 = 场景 Scene
场景 = 很多节点 Node
节点 = 位置/旋转/缩放 + 子节点
组件 Component = 节点身上的能力
脚本 Script = 你自己写的组件
资源 Asset = 图片、音频、字体、预制体等素材
```

## 最终小项目

本教程最后会做一个 **接金币小游戏**：

- 玩家左右移动
- 金币从上方掉落
- 接到金币加分
- 漏掉金币扣生命
- 生命归零游戏结束
- 有开始按钮、分数显示、重新开始

这个项目小，但覆盖了入门最重要的东西：节点、脚本、输入、UI、碰撞、预制体、游戏状态。
