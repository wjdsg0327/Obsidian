# 10-常见错误和排错清单

## 排错第一原则

先看 Console 第一条红色错误。

不要同时处理十个错误。很多后面的错误都是第一个错误引起的。

## 脚本不生效

检查：

- [ ] 脚本有没有保存
- [ ] 编辑器有没有编译完成
- [ ] 脚本有没有挂到节点上
- [ ] 节点 active 是否为 true
- [ ] 类名和装饰器是否正常

## property 拖不上

可能原因：

- 类型不对，比如 `@property(Label)` 却拖 Node
- 脚本有编译错误
- 没保存脚本
- 目标组件不在那个节点上

## 节点看不到

检查：

- [ ] 是否在 Canvas 下
- [ ] 坐标是否跑出屏幕
- [ ] scale 是否为 0
- [ ] active 是否关闭
- [ ] SpriteFrame 是否为空
- [ ] 透明度是否为 0
- [ ] 层级是否被盖住

## 按钮点不了

检查：

- [ ] Button 组件是否存在
- [ ] Click Events 是否绑定
- [ ] 是否有 Canvas / EventSystem 相关设置
- [ ] 是否被其他透明节点挡住
- [ ] 方法是否 public 或可被编辑器识别

## 移动速度异常

检查是否用了：

```ts
speed * deltaTime
```

如果没乘 deltaTime，高帧率机器会跑得更快。

## 碰撞不触发

检查：

- [ ] 两边是否都有 Collider2D
- [ ] 至少一方是否符合触发条件
- [ ] 碰撞体大小是否正确
- [ ] 分组/掩码是否允许碰撞
- [ ] 监听事件是否注册
- [ ] 物理系统是否启用

## 资源丢失 Missing

处理方法：

1. 看资源是否被删除或移动
2. 在 Inspector 重新拖引用
3. 如果 Prefab 实例乱了，重新从 Prefab 拖一个

## TypeScript 常见报错

### Cannot find name xxx

通常是忘记 import。

### Object is possibly null

你定义了可能为 null 的变量，使用前要判断：

```ts
if (!this.scoreLabel) return;
this.scoreLabel.string = '...';
```

### Property xxx has no initializer

给默认值：

```ts
score = 0;
```

或者允许 null：

```ts
player: Node | null = null;
```

## 一个有效的排错流程

```text
看到问题
↓
看 Console 第一条红错
↓
定位是脚本/资源/场景/构建哪类问题
↓
做最小修改
↓
保存并重新运行
↓
确认问题消失，再处理下一个
```

## 给老王的建议

学 Cocos 初期，不要追求架构优雅。先把东西跑起来：

1. 节点能显示
2. 脚本能执行
3. 输入能响应
4. UI 能刷新
5. 游戏能重开

能跑起来以后，再慢慢整理代码。
