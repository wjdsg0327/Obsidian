# 00-学习路线和环境准备

## 先选版本

Cocos Creator 有 2.x 和 3.x 两条常见路线。现在新学建议直接用 **Cocos Creator 3.x**。

推荐原则：

- 新项目：用 3.x 稳定版
- 老项目维护：按项目原版本来
- 微信小游戏/H5：3.x 可以胜任

## 安装

1. 到 Cocos 官网下载 **Cocos Dashboard**
2. 在 Dashboard 里安装 Cocos Creator 编辑器
3. 创建新项目时选择 2D 模板

常见组件：

- **Dashboard**：管理编辑器版本和项目
- **Creator Editor**：真正写游戏的编辑器
- **VS Code**：写 TypeScript 脚本

## 创建第一个项目

建议项目名：`CatchCoins`

模板选：

- 2D Empty 或 2D Core
- 语言：TypeScript

目录大概长这样：

```text
CatchCoins/
├─ assets/          # 主要资源和脚本都在这里
├─ settings/        # 项目设置
├─ package.json
└─ tsconfig.json
```

你一开始重点只看 `assets/`。

## 第一天目标

不要急着学所有功能。第一天只完成三件事：

1. 打开编辑器
2. 新建一个场景
3. 放一个文字 Label 和一个按钮 Button

如果你能做到这一步，说明环境已经通了。

## 编辑器启动后要认识的几个区域

```text
┌───────────────┬────────────────────┬───────────────┐
│ 层级管理器     │ 场景视图 / 游戏视图   │ 属性检查器      │
│ Hierarchy     │ Scene / Game       │ Inspector     │
├───────────────┴────────────────────┴───────────────┤
│ 资源管理器 Assets                                   │
└────────────────────────────────────────────────────┘
```

- **Hierarchy**：当前场景里有哪些节点
- **Scene**：编辑场景
- **Game**：运行预览画面
- **Inspector**：选中节点后改属性
- **Assets**：项目文件、图片、脚本、预制体

## 常见坑

### 坑 1：项目路径有中文或特殊符号

不是一定不行，但初学建议路径简单：

```text
D:/work/cocos/CatchCoins
```

### 坑 2：不知道保存场景

Cocos 里脚本保存不等于场景保存。你改了场景节点，要记得 `Ctrl + S` 保存场景。

### 坑 3：浏览器预览没反应

先看 Console 有没有红色报错。红色报错优先解决第一个，不要同时看十几个。
