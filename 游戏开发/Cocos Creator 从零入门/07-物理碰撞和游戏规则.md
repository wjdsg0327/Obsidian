# 07-物理碰撞和游戏规则

## 碰撞解决什么问题

游戏里经常要判断：

- 玩家有没有碰到金币
- 子弹有没有打到敌人
- 角色有没有踩到地面
- 物体有没有进入某个区域

这就是碰撞检测。

## 2D 碰撞组件

常见组件：

- Collider2D：碰撞体
- BoxCollider2D：矩形碰撞体
- CircleCollider2D：圆形碰撞体
- RigidBody2D：刚体，参与物理运动

入门接金币游戏里，可以先用 Collider2D 做触发检测。

## 开启碰撞监听

大致写法：

```ts
import { _decorator, Component, Collider2D, Contact2DType, IPhysics2DContact } from 'cc';
const { ccclass } = _decorator;

@ccclass('Coin')
export class Coin extends Component {
  onLoad() {
    const collider = this.getComponent(Collider2D);
    if (collider) {
      collider.on(Contact2DType.BEGIN_CONTACT, this.onBeginContact, this);
    }
  }

  onBeginContact(self: Collider2D, other: Collider2D, contact: IPhysics2DContact | null) {
    console.log('碰到了：', other.node.name);
  }
}
```

具体版本 API 可能有细微差异，遇到报错以当前编辑器提示为准。

## 触发器 vs 真实物理

你可以把碰撞分成两类：

1. **触发检测**：只想知道碰到了，比如吃金币
2. **物理模拟**：要弹开、滚动、受重力影响

入门项目优先用触发检测，简单很多。

## 游戏规则放哪里

建议用 `GameManager` 管整体规则：

- 当前分数
- 当前生命
- 游戏是否开始
- 游戏是否结束
- 加分
- 扣血
- 重开

金币自己只负责：

- 下落
- 碰到玩家时通知加分
- 掉出屏幕时通知扣生命

## 金币下落逻辑

```ts
import { _decorator, Component } from 'cc';
const { ccclass, property } = _decorator;

@ccclass('Coin')
export class Coin extends Component {
  @property
  fallSpeed = 250;

  update(dt: number) {
    const pos = this.node.position.clone();
    pos.y -= this.fallSpeed * dt;
    this.node.setPosition(pos);

    if (pos.y < -400) {
      this.node.destroy();
    }
  }
}
```

## 常见错误

### 碰撞没触发

检查：

1. 有没有加 Collider2D
2. 碰撞体大小是否覆盖图片
3. 项目是否启用 2D 物理
4. 碰撞分组/掩码是否允许接触
5. 监听代码是否挂在正确节点

### 碰撞触发多次

接触可能持续多帧。吃金币后立即 `destroy()`，或加 `collected` 标记防重复。

### 规则分散各处

分数在 Coin 里改、生命在 Player 里改、结束在 UI 里改，后面会乱。尽量让 GameManager 管状态。
