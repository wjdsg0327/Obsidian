# 数据库设计说明

> 配套文件：数据库表关系图.html

## 1. 设计原则

- 用户统一放在 `users` 表，教师、学生、家长分别建立扩展表。
- 学校、班级、用户、作业、题库、学情分层设计。
- 作业与题目使用中间表 `assignment_items`，支持一份作业包含多题。
- 批改结果按“提交记录 + 题目”维度记录，方便单题评分与错题归档。
- 错题表保留 `student_id`、`question_id`、`submission_id`，便于查询和追溯。
- 学情统计表可定时汇总，避免每次页面实时计算。

## 2. 核心表分组

### 2.1 用户与组织

- `schools`：学校表
- `users`：统一用户表
- `teachers`：教师扩展表
- `students`：学生扩展表
- `parents`：家长扩展表
- `parent_students`：家长与学生绑定关系
- `classes`：班级表

### 2.2 作业与批改

- `assignments`：作业表
- `assignment_items`：作业题目表
- `submissions`：作业提交记录表
- `grading_records`：批改记录表

### 2.3 题库与知识体系

- `questions`：题目表
- `knowledge_points`：知识点表
- `syllabi`：考纲表
- `textbooks`：教材表
- `workbooks`：教辅表
- `words`：单词表

### 2.4 学情与统计

- `learning_stats`：学情数据表
- `wrong_questions`：错题表
- `teacher_quality_stats`：教学质量统计表
- `class_compare_stats`：班级对比数据表

### 2.5 系统与日志

- `teacher_quotas`：教师限额表
- `operation_logs`：操作记录表
- `system_logs`：系统日志表

## 4. 建议索引

- `users(school_id, role, status)`
- `classes(school_id, grade, subject, course_system, exam_board)`
- `students(class_id, student_no)`
- `assignments(class_id, teacher_id, status, due_at)`
- `submissions(assignment_id, student_id, status)`
- `grading_records(submission_id, question_id)`
- `questions(course_system, subject, exam_board, knowledge_point_id, difficulty, type)`
- `wrong_questions(student_id, question_id, mastered_at)`
- `learning_stats(student_id, class_id, knowledge_point_id, stat_date)`

## 5. 后续待确认

- 是否需要多班级任课教师关系表：`class_teachers`
- 是否允许一个学生同时属于多个班级：如是，需要 `class_students`
- 是否需要作业发布到“指定学生/分组”：如是，需要 `assignment_targets`
- 是否需要保存 AI 调用明细：建议增加 `ai_call_logs`
- 是否需要对象存储文件表：建议增加 `files`


## 6. 审查后补充的关键表

为解决作业发布范围、文件管理、通知、教学统计与多教师任课关系，补充以下表：

| 表名 | 用途 |
|---|---|
| `class_teachers` | 支持一个班级多个任课教师，避免仅靠班主任字段表达教学关系 |
| `assignment_targets` | 支持作业发布到全班、指定分组、指定学生 |
| `files` | 统一管理提交图片、PDF、课件、录音、题库导入文件等文件元数据 |
| `notifications` | 作业提醒、批改完成、限额提醒、系统通知 |
| `teacher_quality_stats` | 教师教学质量统计 |
| `class_compare_stats` | 同年级/同学科/同考局班级横向对比统计 |

## 7. 安全口径修正

密码存储建议使用 `Argon2id`，其次使用 `bcrypt`。不建议自行实现 `SHA-256 + 盐值` 作为密码哈希方案。

## 8. 数据权限实现要求

- 所有查询必须在后端附加学校、班级、教师、学生或家长绑定关系过滤。
- 数据导出、批量操作、改分、题库审核必须写 `operation_logs`。
- AI 调用应通过匿名学生 ID 或业务 ID，避免向模型供应商传递不必要个人信息。
