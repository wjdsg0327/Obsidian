# 后端模块划分

## 1. 模块总览

```text
backend
├─ auth          登录、Token、权限
├─ user          用户、教师、学生、家长
├─ school        学校、学期、系统配置
├─ class         班级、班级学生、任课关系
├─ question      题库、知识点、考纲、教材教辅
├─ assignment    作业、作业题目、发布范围
├─ submission    学生提交、文件上传
├─ grading       OCR、AI批改、人工复核
├─ analytics     学情统计、报告生成
├─ quota         教师限额、AI调用成本
├─ notification  消息通知
├─ log           操作日志、系统日志
└─ admin         管理端聚合接口
```

## 2. Auth 模块

职责：

- 登录、退出。
- JWT 签发和刷新。
- 当前用户信息。
- 权限中间件。

核心接口：

- `POST /auth/login`
- `POST /auth/logout`
- `GET /auth/me`

## 3. User 模块

职责：

- 用户统一管理。
- 教师/学生/家长扩展信息。
- 批量导入。
- 启用/禁用账号。
- 重置密码。

核心表：

- `users`
- `teachers`
- `students`
- `parents`
- `parent_students`

## 4. School / Class 模块

职责：

- 学校基础信息。
- 班级创建、编辑、归档。
- 班级学生管理。
- 后续支持任课教师关系。

核心表：

- `schools`
- `classes`
- `students`

建议后续表：

- `class_teachers`
- `class_students`

## 5. Question 模块

职责：

- 题目 CRUD。
- Excel/Word/PDF 导入。
- AI 自动打标结果保存。
- 人工审核。
- 知识点树、考纲、教材、教辅管理。

核心表：

- `questions`
- `knowledge_points`
- `syllabi`
- `textbooks`
- `workbooks`
- `words`

## 6. Assignment 模块

职责：

- 作业草稿。
- 手动选题。
- AI 推荐题目。
- 发布作业。
- 作业复制。
- PDF 生成。

核心表：

- `assignments`
- `assignment_items`

后续建议：

- `assignment_targets` 支持指定学生/分组。

## 7. Submission 模块

职责：

- 学生提交作业。
- 多文件上传和排序。
- 提交状态管理。
- 逾期规则判断。

核心表：

- `submissions`

## 8. Grading 模块

职责：

- OCR 识别。
- AI 批改。
- 教师复核。
- 分数修改。
- 错题生成。

核心表：

- `grading_records`
- `wrong_questions`
- `ai_call_logs`

## 9. Analytics 模块

职责：

- 班级学情统计。
- 学生学情统计。
- 知识点掌握率。
- 报告导出。
- AI 学情总结。

核心表：

- `learning_stats`
- `teacher_quality_stats`
- `class_compare_stats`

## 10. Quota 模块

职责：

- 教师使用额度。
- AI 调用次数统计。
- 成本统计。
- 阈值提醒。

核心表：

- `teacher_quotas`
- `ai_call_logs`

## 11. Log 模块

职责：

- 操作日志。
- 系统日志。
- 登录日志。
- 数据导出审计。

核心表：

- `operation_logs`
- `system_logs`

## 12. 后端开发顺序建议

1. Auth + User。
2. School + Class。
3. Question。
4. Assignment。
5. Submission。
6. Grading。
7. Analytics。
8. Quota + Log。
9. Admin 聚合看板。

## 13. 技术实现建议

- 所有写操作使用事务。
- 数据权限在 Repository/Query 层统一追加过滤条件。
- AI 批改、PDF 生成、批量导入用异步任务队列。
- 文件上传先落对象存储，再写数据库。
- 关键接口加幂等键，避免重复提交。
