# 业务状态机

> 本文档定义作业、提交、批改、题目审核、AI 任务等关键业务状态。前端展示、后端接口、测试用例均应使用统一状态。

## 1. 作业状态 assignment.status

| 状态 | 含义 | 可执行操作 | 下一状态 |
|---|---|---|---|
| draft | 草稿 | 编辑、删除、发布 | published / archived |
| published | 已发布 | 查看提交、关闭、归档 | closed / archived |
| closed | 已截止/关闭 | 查看、统计、归档 | archived |
| archived | 已归档 | 查看历史 | - |

### 1.1 作业状态流转

```text
draft → published → closed → archived
  └──────────────→ archived
```

### 1.2 规则

- 草稿作业学生不可见。
- 已发布作业学生可见。
- 到截止时间后可以自动进入 closed，或继续保持 published 但提交按逾期规则处理。
- archived 作业不允许新增提交，只允许查看历史。

## 2. 提交状态 submissions.status

| 状态 | 含义 | 可执行操作 | 下一状态 |
|---|---|---|---|
| not_submitted | 未提交 | 学生提交 | submitted |
| submitted | 已提交 | 触发 OCR/AI 批改 | grading |
| grading | 批改中 | 等待 AI 或教师处理 | reviewed / manual_review |
| manual_review | 待人工复核 | 教师复核 | reviewed / returned |
| reviewed | 已批改 | 学生查看、订正 | corrected |
| returned | 已打回 | 学生重新提交 | submitted |
| corrected | 已订正 | 教师查看 | reviewed |

### 2.1 提交流转

```text
not_submitted → submitted → grading → reviewed → corrected
                         └→ manual_review → reviewed
                         └→ returned → submitted
```

## 3. 批改记录状态 grading_records.review_status

| 状态 | 含义 |
|---|---|
| ai_pending | 等待 AI 批改 |
| ai_completed | AI 已批改 |
| manual_required | 需要人工复核 |
| teacher_confirmed | 教师已确认 |
| appealed | 学生/教师提出复核争议 |

## 4. 题目审核状态 questions.review_status

| 状态 | 含义 | 说明 |
|---|---|---|
| draft | 导入草稿 | 未进入审核 |
| ai_tagged | AI 已打标 | 等待人工检查 |
| pending_review | 待审核 | 管理员审核队列 |
| approved | 已通过 | 可用于公共组题 |
| rejected | 已驳回 | 需修改后重新提交 |
| archived | 已归档 | 不再用于新作业 |

### 4.1 题目审核流转

```text
draft → ai_tagged → pending_review → approved
                              └────→ rejected → draft
approved → archived
```

## 5. AI 任务状态 ai_task.status

| 状态 | 含义 |
|---|---|
| queued | 已入队 |
| running | 执行中 |
| success | 成功 |
| failed | 失败 |
| retrying | 重试中 |
| cancelled | 已取消 |
| manual_fallback | 转人工处理 |

## 6. 文件状态 files.status

| 状态 | 含义 |
|---|---|
| uploading | 上传中 |
| uploaded | 已上传 |
| processing | 处理中，如 OCR、转码、生成缩略图 |
| ready | 可用 |
| failed | 处理失败 |
| deleted | 已删除或归档不可用 |

## 7. 通知状态 notifications.status

| 状态 | 含义 |
|---|---|
| unread | 未读 |
| read | 已读 |
| dismissed | 已忽略 |

## 8. 逾期规则 late_rule

| 值 | 含义 |
|---|---|
| allow_late | 允许逾期提交，仅标记逾期 |
| deny_late | 截止后禁止提交 |
| penalty_late | 允许逾期但按比例扣分 |

## 9. 测试关注点

- 每个状态必须有明确入口和出口。
- 非法状态流转必须被后端拒绝。
- 状态变更必须写操作日志。
- 状态变更后相关统计要刷新或进入异步汇总。
