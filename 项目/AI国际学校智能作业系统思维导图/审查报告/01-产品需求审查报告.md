# 01-产品需求审查报告

## 1. 总体评分

**评分：82 / 100**

当前产品文档已经覆盖需求、功能清单、页面清单、用户故事、MVP 范围、权限矩阵和排期，足以进入初步产品评审。但仍存在一些口径不统一、MVP 边界略宽、家长端/移动端范围不够明确的问题。

## 2. 主要问题

### 2.1 MVP 课程范围口径不完全一致

- 原需求文档中提到一期完成 DSE、A-Level、GCSE、AP 化学相关内容。
- 后续 MVP 文档多处写“优先 A-Level / IGCSE”。
- 需要统一为一个研发口径，否则排期和题库准备会失控。

**建议统一口径：**

- MVP：A-Level / IGCSE 化学。
- 一期扩展：DSE / AP / GCSE 化学。
- 二期扩展：IB 与更多学科。

### 2.2 家长端范围需要收缩

家长端文档已有作业、报告、沟通功能，但 MVP 中又说家长端可后置。需要明确：

- MVP 是否做家长登录？
- 是否只做报告分享链接？
- 家校沟通是否后置？

建议 MVP 只保留“家长查看报告”的只读能力，家校沟通后置。

### 2.3 教师是否可建班级存在不确定

权限矩阵中教师可创建本人班级，但也标注“建议”。这会影响后台流程和数据权限。

建议：MVP 中班级由学校管理员创建；教师只管理自己班级学生和作业。这样权限更稳。

### 2.4 页面原型说明缺少关键状态

目前页面清单覆盖完整，但缺少状态流转：

- 作业状态：draft / published / closed / archived。
- 提交状态：not_submitted / submitted / grading / reviewed / returned。
- 题目审核状态：draft / ai_tagged / pending_review / approved / rejected / archived。

### 2.5 用户故事还缺少异常场景

已有主流程验收，但缺少：

- 作业逾期。
- AI 批改失败。
- 文件上传失败。
- 教师限额不足。
- 题库导入部分失败。

## 3. 遗漏点

1. 业务状态机文档。
2. 数据字典 / 枚举值说明。
3. 文件上传规则。
4. 消息通知规则。
5. 报告模板说明。
6. 运营后台或超管后台的租户边界说明。

## 4. 冲突点

| 冲突 | 涉及文档 | 建议 |
|---|---|---|
| MVP 课程范围不一致 | 原需求、MVP范围、产品功能清单 | 统一为 A-Level / IGCSE 化学 |
| 教师是否能建班级 | 权限矩阵、页面清单 | MVP 由管理员建班，教师管理班级 |
| 家长端是否 MVP | MVP范围、页面清单、用户故事 | MVP 只读查看报告，沟通后置 |
| 密码算法表述 | 原需求、部署安全 | 用 Argon2/bcrypt 替代 SHA-256+盐 |
