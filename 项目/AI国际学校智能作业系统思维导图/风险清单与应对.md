# 风险清单与应对

## 1. 高风险项

| 风险 | 影响 | 概率 | 应对策略 |
|---|---|---:|---|
| AI 批改准确率不足 | 影响核心卖点和验收 | 高 | 客观题规则引擎优先；主观题教师复核；设置置信度阈值 |
| OCR 识别手写不稳定 | 影响批改效率 | 高 | 上传质量检测；低置信度人工处理；先支持清晰拍照/扫描 |
| 题库资料不完整 | 智能组题效果差 | 高 | 先聚焦 A-Level/IGCSE 化学；建立导入与审核流程 |
| 版权不清晰 | 合规风险 | 中高 | 每题记录版权来源；未确认版权只进个人题库或待审核 |
| MVP 范围膨胀 | 延期 | 高 | 严格 P0/P1/P2；家长端和复杂分析后置 |
| 数据权限遗漏 | 数据泄露 | 中高 | 后端统一数据范围控制；权限矩阵验收 |

## 2. 技术风险

| 风险 | 应对 |
|---|---|
| 批量上传大文件慢 | 分片上传、异步处理、进度提示 |
| AI 调用超时 | 队列化、重试、人工兜底 |
| 数据库统计慢 | 建立统计表和索引，定时汇总 |
| PDF 生成耗时 | 异步生成，完成后通知 |
| 并发提交高峰 | 限流、队列、对象存储直传 |

## 3. 产品风险

| 风险 | 应对 |
|---|---|
| 教师学习成本高 | 核心操作 3 步内，新手引导，模板化作业 |
| 学生上传不规范 | 上传页提供示例图和自动裁剪提示 |
| 家长过度关注单次成绩 | 报告强调趋势和建议 |
| 教师不信任 AI 批改 | 展示评分依据，允许快速复核和修改 |

## 4. 合规风险

| 风险 | 应对 |
|---|---|
| 未成年人个人信息保护 | 最小化采集、加密存储、访问审计 |
| 数据导出泄露 | 权限控制、水印、日志记录 |
| AI 服务传输隐私数据 | 匿名化学生信息，只传必要内容 |
| 备份数据泄露 | 备份加密、权限隔离、定期清理 |

## 5. 项目管理风险

| 风险 | 应对 |
|---|---|
| 需求反复 | 每周评审，变更进入 backlog |
| 测试时间不足 | 从 W3 开始持续测试，W11 只做回归 |
| 交付标准不明确 | 使用用户故事和验收标准逐项确认 |
| 外部 AI 供应商不稳定 | 适配层设计，预留供应商切换能力 |

## 6. 风险优先处理建议

1. 先做 AI 批改 PoC，验证 OCR 和评分准确率。
2. 先确定题库资料来源和版权边界。
3. 先锁定 MVP 只做 Web 端。
4. 先完成权限矩阵并落实到接口层。
5. 先建立 AI 调用日志和成本统计。
