# 统一口径与假设

> 本文档用于统一 AI 国际学校智能作业系统项目文档中的范围、术语、技术假设和优先级口径。后续产品、研发、测试、交付均以本文为准。

## 1. 项目定位

AI 国际学校智能作业系统是一套面向国际课程学校的智能作业平台，核心目标是打通：

```text
教师布置作业 → 学生提交作业 → AI/教师批改 → 错题沉淀 → 学情分析 → 教学改进
```

系统重点不是单纯题库，也不是单纯 AI 批改工具，而是围绕国际课程教学场景的作业闭环平台。

## 2. MVP 范围统一口径

### 2.1 MVP 必做范围

MVP 统一定义为：

- **课程体系：A-Level / IGCSE**
- **优先学科：Chemistry / 化学**
- **核心用户：学校管理员、教师、学生**
- **家长端：只做只读预留或报告查看能力，不做完整家校沟通**
- **核心闭环：智能组题、作业发布、学生提交、AI 批改、教师复核、错题集、基础学情分析**

### 2.2 一期扩展范围

一期扩展可以加入：

- DSE / AP / GCSE 化学题库与作业流程
- 更完整教师限额管理
- 报告导出
- 家长查看报告
- PDF 生成增强

### 2.3 二期扩展范围

二期扩展可以加入：

- IB 课程体系
- 更多学科
- 家校沟通
- 微信小程序 / 公众号
- 教师教学质量排名
- 班级横向对比高级分析
- 复杂主观题更深度 AI 批改

## 3. 角色权限口径

### 3.1 MVP 角色

| 角色 | MVP 是否重点 | 说明 |
|---|---:|---|
| 超级管理员 | 是 | 平台和多学校管理 |
| 学校管理员 | 是 | 本校账号、班级、题库、限额、统计 |
| 教师 | 是 | 作业与学情核心使用者 |
| 学生 | 是 | 作业提交和反馈查看 |
| 家长 | 否/弱化 | MVP 只读预留，完整功能后置 |

### 3.2 教师建班规则

MVP 统一规则：

- 班级由学校管理员创建。
- 教师可管理自己所教班级中的学生、作业和学情。
- 教师如需新建班级，先由学校管理员操作；后续可增加“教师申请建班”。

## 4. 技术假设

| 项目 | 假设 |
|---|---|
| 前端 | Web 响应式优先 |
| 移动端 | 学生端适配手机浏览器，暂不做原生 App |
| 后端 | REST API，后续可扩展异步任务队列 |
| 数据库 | PostgreSQL 优先，MySQL 可适配 |
| 文件存储 | 对象存储或 MinIO，数据库只存文件元数据 |
| AI 调用 | 通过后端 AI Service Adapter，不由前端直接调用供应商 |
| AI 批改 | AI 辅助，教师可复核，低置信度必须人工确认 |
| 密码安全 | Argon2id 优先，bcrypt 可接受，不建议手写 SHA-256+盐 |

## 5. 数据权限口径

- 超级管理员：全平台数据。
- 学校管理员：本校数据。
- 教师：所教班级和相关学生数据。
- 学生：本人作业和学情数据。
- 家长：绑定子女数据。

所有数据权限必须由后端强制校验，前端菜单隐藏不能作为安全边界。

## 6. AI 验收口径

MVP 中 AI 结果不是无条件最终结果：

- 客观题可自动出最终建议分。
- 简单填空/计算题可自动出建议分，但低置信度需人工复核。
- 复杂论述、画图、实验题默认人工批改。
- AI 批改结果必须记录置信度、模型、调用时间和成本。

## 7. 文档优先级口径

- P0：影响 MVP 闭环、权限、安全、数据结构、验收的文档。
- P1：影响效率和体验，但不阻断 MVP 的文档。
- P2：后续商业化、规模化、精细化运营文档。
