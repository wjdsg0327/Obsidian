# 部署运维方案

## 1. 推荐部署架构

- 前端：Nginx / CDN
- 后端：API 服务，支持水平扩展
- 数据库：MySQL 或 PostgreSQL
- 缓存：Redis
- 文件存储：对象存储或本地 MinIO
- AI 服务：OCR、批改、组题、学情总结接口
- 日志：应用日志、操作日志、AI 调用日志
- 监控：服务状态、接口耗时、数据库连接、AI 调用成本

## 2. 环境划分

| 环境 | 用途 |
|---|---|
| dev | 开发自测 |
| test | 测试人员验证 |
| staging | 上线前预发布 |
| prod | 正式生产 |

## 3. 配置项

- 数据库连接
- Redis 连接
- 文件存储路径或对象存储 Bucket
- AI 服务 API Key
- OCR 服务配置
- JWT 密钥
- HTTPS 证书
- 日志级别
- 上传文件大小限制
- 备份策略

## 4. 数据备份

- 每日凌晨全量备份数据库
- 重要表支持增量备份
- 备份保留 30 天
- 上传文件与数据库分开备份
- 每月至少做一次恢复演练

## 5. 安全要求

- 全站 HTTPS
- 管理端限制弱密码
- Token 设置过期时间
- 登录失败次数限制
- 文件上传类型校验
- 所有用户关键操作写入操作日志
- AI 服务 Key 不得写入前端代码
- 数据导出需要权限控制和日志记录

## 6. 监控指标

- API 成功率和错误率
- 平均响应时间、P95 响应时间
- 数据库慢查询
- Redis 命中率
- 文件上传失败率
- AI 调用次数、失败率、平均耗时、成本
- 作业批改队列长度
- 磁盘空间和备份状态

## 7. 发布流程

1. 开发环境自测通过
2. 提交测试环境
3. 测试执行回归用例
4. 预发布环境验证数据库迁移
5. 备份生产数据库
6. 灰度发布或低峰期发布
7. 发布后观察日志和监控
8. 异常时按回滚方案处理

## 8. 回滚方案

- 保留上一版本镜像或代码包
- 数据库迁移脚本需要提供 down 脚本或备份恢复点
- 发布失败时恢复上一版本服务
- 若涉及数据结构变化，优先使用兼容性迁移，避免不可逆变更

## 9. 运维待确认

- 生产服务器规格
- 是否使用 Docker Compose / Kubernetes
- 数据库使用 MySQL 还是 PostgreSQL
- 文件存储使用本地 MinIO 还是云对象存储
- AI 服务供应商与成本预算
- 是否需要学校独立租户隔离
