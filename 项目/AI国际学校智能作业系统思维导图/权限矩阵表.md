# 权限矩阵表

> 适用范围：AI 国际学校智能作业系统 MVP 与后续扩展。  
> 角色：超级管理员、学校管理员、教师、学生、家长。

## 1. 权限设计原则

- **最小权限原则**：用户只拥有完成当前角色工作所需的最小权限。
- **学校数据隔离**：学校管理员、教师、学生、家长默认只能访问本校或本人相关数据。
- **角色 + 数据范围双重控制**：不仅判断能不能访问功能，还要判断能访问哪些学校、班级、学生、作业。
- **关键操作留痕**：账号、题库、作业发布、批改修改、数据导出等操作必须写入操作日志。
- **MVP 优先简单可控**：先做固定角色权限，后续再扩展自定义权限组。

## 2. 数据范围定义

| 数据范围 | 含义 |
|---|---|
| 全平台 | 所有学校、所有用户、所有数据 |
| 本校 | 当前用户所属学校的数据 |
| 所教班级 | 教师任课或班主任负责的班级 |
| 本班/本人 | 学生所在班级或本人数据 |
| 绑定子女 | 家长绑定的子女数据 |

## 5. API 权限控制建议

- 后端每个接口必须配置：`requiredRole` + `dataScope`。
- 数据查询必须带学校、班级或用户过滤条件，不允许仅靠前端隐藏菜单。
- 教师访问学生数据时，必须校验学生是否属于教师任课班级。
- 家长访问学生数据时，必须校验 `parent_students` 绑定关系。
- 数据导出、批量删除、改分、题库审核等敏感操作必须写操作日志。
