# 测试与验收方案

## 1. 测试目标

验证 AI 国际学校智能作业系统 MVP 是否满足核心业务闭环：账号管理、班级管理、题库管理、作业发布、学生提交、AI/教师批改、学情分析。

## 2. 测试范围

- 功能测试
- 权限测试
- 文件上传测试
- AI 批改测试
- 学情统计测试
- 性能测试
- 安全测试
- 兼容性测试
- 用户验收测试

## 3. 功能验收用例

### 3.1 账号与班级

| 编号 | 场景 | 验收标准 |
|---|---|---|
| U-001 | 管理员创建教师账号 | 教师可登录并进入教师端 |
| U-002 | 批量导入学生 | 学生账号生成，班级关联正确 |
| U-003 | 班级归档 | 历史作业和学情数据仍可查询 |
| U-004 | 角色权限 | 学生不能访问教师端和管理员端 |

### 3.2 题库

| 编号 | 场景 | 验收标准 |
|---|---|---|
| Q-001 | 导入题目 | 题干、答案、解析、标签保存正确 |
| Q-002 | 题目筛选 | 可按课程、学科、知识点、难度、题型筛选 |
| Q-003 | 题目审核 | 未审核题目不能进入公开题库 |

### 3.3 作业

| 编号 | 场景 | 验收标准 |
|---|---|---|
| A-001 | 教师创建作业草稿 | 草稿可保存和继续编辑 |
| A-002 | AI 推荐题目 | 推荐结果符合筛选条件 |
| A-003 | 发布作业 | 学生端能看到待完成作业 |
| A-004 | 截止时间 | 超时提交按规则处理 |
| A-005 | PDF 生成 | 学生版和教师版内容正确 |

### 3.4 提交与批改

| 编号 | 场景 | 验收标准 |
|---|---|---|
| S-001 | 学生上传图片 | 图片可预览并归档到提交记录 |
| S-002 | AI OCR | 能识别主要文字和答案区域 |
| S-003 | AI 批改 | 自动生成得分、对错、解析 |
| S-004 | 教师复核 | 教师可修改分数和评语 |
| S-005 | 错题生成 | 错题自动进入学生错题集 |

### 3.5 学情分析

| 编号 | 场景 | 验收标准 |
|---|---|---|
| L-001 | 班级学情 | 显示平均分、完成率、知识点掌握率 |
| L-002 | 学生学情 | 显示得分趋势和薄弱知识点 |
| L-003 | 报告导出 | PDF/Excel 可生成并下载 |

## 4. 性能验收标准

- 页面首次加载 ≤ 2 秒
- 页面操作响应 ≤ 1 秒
- 单份作业 AI 批改 ≤ 10 秒
- 50 份作业批量批改 ≤ 3 分钟
- 数据库查询响应 ≤ 500 毫秒
- 支持 1000 在线用户，200 并发用户
- 100MB 文件上传支持断点续传或失败重试

## 5. 安全验收标准

- 密码不得明文存储
- 所有接口必须鉴权
- 不同角色只能访问权限范围内数据
- 防 SQL 注入、XSS、CSRF
- 操作日志记录关键动作
- 学生个人信息和学习数据按要求加密或脱敏

## 6. 兼容性验收

- Chrome、Edge、Safari 主流版本
- 桌面端、平板、手机响应式布局
- 图片上传支持 JPG、PNG、HEIC 转换策略
- PDF 上传和预览可用

## 7. 交付验收材料

- 需求文档
- 数据库设计说明
- 接口文档
- 测试报告
- 部署说明
- 管理员/教师/学生操作说明
- 线上环境账号与权限清单
